## 📈 Evaluation Methodology

### Dual-Track Validation

We evaluate each skill across **12 dimensions** split into two tracks:

#### Text Quality Track (50% of overall score)

| Dimension | Weight | Description |
|-----------|--------|-------------|
| System Prompt | 20% | Clear role definition, identity, expertise |
| Domain Knowledge | 20% | Accurate frameworks, standards, methodologies |
| Workflow | 20% | Clear processes, steps, actionable guidance |
| Error Handling | 15% | Risk documentation, pitfalls, mitigations |
| Examples | 15% | Scenario examples, test cases, demonstrations |
| Metadata | 10% | Complete frontmatter, description, tags |

#### Runtime Quality Track (50% of overall score)

| Dimension | Weight | Description |
|-----------|--------|-------------|
| Role Immersion | 20% | Maintains character, professional tone |
| Framework Execution | 20% | Follows decision frameworks consistently |
| Output Actionability | 20% | Concrete, actionable outputs |
| Knowledge Accuracy | 15% | Real standards, correct terminology |
| Conversation Stability | 15% | Handles multi-turn, consistent responses |
| Resilience | 10% | Handles errors and edge cases gracefully |

### Certification Criteria

A skill is **CERTIFIED** when it meets all of:

- ✅ Text Score ≥ 8.0
- ✅ Runtime Score ≥ 8.0
- ✅ Variance < 1.0 (Text and Runtime scores are balanced)
- ✅ All 12 dimensions ≥ 6.0
