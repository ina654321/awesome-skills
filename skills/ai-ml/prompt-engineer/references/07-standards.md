# Standards & Reference

## 7.1 ML Lifecycle

### Stages
| Stage | Deliverable | Key Activities |
|-------|-------------|----------------|
| Design | Problem framing | Define metrics, baseline |
| Data | Dataset | Collect, validate, label |
| Model | Trained model | Train, tune, evaluate |
| Deploy | Serving system | Serve, monitor |
| Monitor | Performance | Detect drift, retrain |

### Model Selection
| Problem Type | Algorithms |
|-------------|------------|
| Classification | Logistic, RF, XGBoost, Neural Nets |
| Regression | Linear, Ridge, Random Forest |
| Clustering | K-means, DBSCAN, Hierarchical |
| NLP | Transformers, BERT, GPT |
| Vision | CNN, ResNet, ViT |

## 7.2 MLOps Best Practices

- Version control (data, code, model)
- Experiment tracking
- CI/CD for ML
- Feature store
- Model registry
- Automated retraining
- Monitoring and alerting

## 7.3 LLM Evaluation

| Metric | What it Measures |
|--------|-----------------|
| Perplexity | Language fluency |
| BLEU | N-gram overlap with reference |
| ROUGE | Summary overlap |
| GPT-judge | Human preference alignment |
