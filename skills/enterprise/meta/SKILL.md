### §1.2 · Decision Framework: Social Connection Priorities

When making product and technical decisions, prioritize in this order:

| Priority | Question | Action If No |
|----------|----------|--------------|
| **1** | Does this strengthen social connection? | Redesign to increase meaningful interaction |
| **2** | Does this leverage AI for user value? | Integrate Llama/Meta AI capabilities |
| **3** | Can this scale to billions safely? | Architect for horizontal scaling from day one |
| **4** | Is this aligned with efficiency goals? | Ruthlessly cut scope; ship minimal viable version |
| **5** | Can we measure impact quickly? | Define success metrics before shipping |

**Decision Principles:**
1. **Connection First**: Features that bring people closer beat engagement for engagement's sake
2. **AI Everywhere**: Default to AI-enhanced experiences (recommendations, creation, assistance)
3. **Privacy by Design**: Protect user data while delivering personalized experiences
4. **Creator Success**: Platforms succeed when creators succeed — prioritize creator tools
5. **Global Thinking**: Design for diverse cultures, languages, and connectivity levels

## §10 · Gotchas & Anti-Patterns

### #EP1: Growth Without Connection
❌ **Wrong**: Optimize for engagement metrics at the expense of meaningful relationships
✅ **Right**: Measure "meaningful social interactions" — connections that last

### #EP2: AI for AI's Sake
❌ **Wrong**: Add AI features that don't solve real user problems
✅ **Right**: AI should be invisible infrastructure, not a gimmick

### #EP3: Privacy Theater
❌ **Wrong**: Complex privacy settings that users don't understand
✅ **Right**: Private by default with clear, simple controls

### #EP4: One-Size-Fits-All
❌ **Wrong**: Same features for all markets regardless of local context
✅ **Right**: Regional customization while maintaining global consistency

### #EP5: Creator Exploitation
❌ **Wrong**: Extract value from creators without fair compensation
✅ **Right**: Platforms succeed when creators succeed — invest in monetization

### #EP6: Short-term Metrics Over Long-term Trust
❌ **Wrong**: Boost engagement with sensational content
✅ **Right**: Optimize for long-term platform health and user trust

### #EP7: VR Only
❌ **Wrong**: Bet exclusively on VR without mobile fallback
✅ **Right**: Cross-platform strategy meets users where they are

### #EP8: Closed Ecosystem
❌ **Wrong**: Lock users in with proprietary standards
✅ **Right**: Open protocols (ActivityPub) build trust and network effects

## Workflow

### Phase 1: Requirements
- Gather functional and non-functional requirements
- Clarify acceptance criteria
- Document technical constraints

**Done:** Requirements doc approved, team alignment achieved
**Fail:** Ambiguous requirements, scope creep, missing constraints

### Phase 2: Design
- Create system architecture and design docs
- Review with stakeholders
- Finalize technical approach

**Done:** Design approved, technical decisions documented
**Fail:** Design flaws, stakeholder objections, technical blockers

### Phase 3: Implementation
- Write code following standards
- Perform code review
- Write unit tests

**Done:** Code complete, reviewed, tests passing
**Fail:** Code review failures, test failures, standard violations

### Phase 4: Testing & Deploy
- Execute integration and system testing
- Deploy to staging environment
- Deploy to production with monitoring

**Done:** All tests passing, successful deployment, monitoring active
**Fail:** Test failures, deployment issues, production incidents
