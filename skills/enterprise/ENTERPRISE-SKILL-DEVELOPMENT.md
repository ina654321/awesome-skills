## 🔧 开发流程 (标准SOP)

### Phase 1: 信息收集 (1-2天)
```
1. 搜索公开招聘信息 (Job Postings)
   - LinkedIn, 官方招聘页面
   - 关键词: "面试", "文化", "价值观"

2. 收集面试经验分享
   - 一亩三分地, 牛客, 知乎
   - 应届生论坛, Glassdoor

3. 研究企业文化文档
   - 官方文化宣言 (如Netflix Culture Deck)
   - 创始人/高管访谈
   - 相关书籍 (如Working Backwards)

4. 提炼核心方法论
   - 3-5个决策框架
   - 核心价值观清单
   - 面试考察重点
```

### Phase 2: 框架设计 (半天)
```
1. 确定决策框架
   - 设计5 Gates格式
   - 明确每个gate的评判标准

2. 设计思维模型
   - 核心工作法 (如Working Backwards)
   - 决策工具 (如One-way vs Two-way door)

3. 构建三层架构
   - Culture: 核心价值观
   - Methodology: 方法论/框架
   - Tools: 实操工具

4. 规划双重视角
   - Job Seeker: 面试准备
   - Practitioner: 日常工作
```

### Phase 3: Skill编写 (1天)
```
1. 基于TEMPLATE-enterprise.md
2. 填充16个标准章节
   - 特别关注: §5 Toolkit, §8 Scenarios, §9 Anti-patterns
3. 编写3+场景示例
   - 面试场景
   - 工作决策场景
   - 跨团队协作场景
4. 添加反模式 (8个Enterprise anti-patterns)
```

### Phase 4: 质量验证 (半天)
```
1. 运行skill-scorer.sh
   - SDI ≥ 2.0 (Good), ≥ 3.5 (Exceptional)
   - 16个章节完整
   - 4+风险项
   - 2+场景示例

2. 结构检查
   - 章节符号统一 (## 1. 格式)
   - URL路径正确
   - YAML元数据完整

3. 内容审查
   - 三层架构清晰
   - 双重视角覆盖
   - 反模式实用

4. 自评打分
   - 使用weighted rubric
   - 记录评分依据
```

### Phase 5: 发布 (半天)
```
1. 更新CATALOG.md
2. 更新README.md (如需要)
3. 更新总Skill数量
4. 记录LESSONS_LEARNED.md
```

## 📋 质量标准 (Enterprise Skill)

### SDI (Structure Density Index)
```
Formula: (tables × 3 + code_blocks × 2 + lists) / (lines / 100)

Target:
- ≥ 2.0: Good
- ≥ 3.5: Exceptional

Tesla Engineer: ~3.2
Amazon Engineer: ~28.0 (大量表格)
Netflix Engineer: ~29.6
```

### Required Elements
| Element | Minimum | Tesla | Amazon | Netflix |
|---------|---------|-------|--------|---------|
| Tables | 5 | 12 | 19 | 18 |
| Code Blocks | 1 | 2 | 2 | 3 |
| List Items | 20 | 45 | 93 | 106 |
| Risk Entries | 4 | 5 | 20 | 24 |
| Scenarios | 2 | 3 | 3 | 3 |
| Anti-patterns | 3 | 8 | 8 | 8 |

### 评分标准 (Weighted Rubric)
| Category | Weight | Description |
|----------|--------|-------------|
| System Prompt | 20% | Role definition, core directives, response format |
| Domain Knowledge | 25% | Frameworks, tools, standards |
| Workflow | 15% | Phase/step structure, checkpoints |
| Risk | 10% | Coverage, specificity, mitigation |
| Examples | 20% | Scenario relevance, depth, variety |
| Metadata | 10% | Completeness, accuracy, installability |

## 💡 最佳实践

### 1. 信息来源优先级
1. **官方来源** (最高): 企业文化宣言, 招聘页面
2. **书籍**: Working Backwards (Amazon), No Rules Rules (Netflix)
3. **高管访谈**: YouTube, Podcasts
4. **员工分享**: 知乎, 牛客, 一亩三分地
5. **新闻报道**: 商业媒体分析

### 2. 反模式设计原则
- 每个反模式对应一个常见误解
- 提供❌错误示例和✅正确做法
- 引用企业官方立场 (如有)

### 3. 双重视角平衡
- Job Seeker: 面试技巧, 准备清单
- Practitioner: 工作方法, 决策框架
- 比例: 约30% : 70%

### 4. 场景示例选择
- **面试场景**: 行为面试问题回答
- **工作场景**: 日常技术/产品决策
- **协作场景**: 跨团队沟通/冲突

## 🎓 经验教训 (Lessons Learned)

### ✅ 成功经验
1. **搜索关键词优化**: 使用"面试", "文化", "领导力原则"等词提高命中率
2. **三层架构清晰**: 帮助用户理解"理念-方法-工具"的层次
3. **双重视角设计**: 同时满足求职者和在职者的需求
4. **反模式实用**: 纠正常见误解, 提高Skill的实用性

### ⚠️ 常见陷阱
1. **信息过载**: 避免堆砌过多细节, 聚焦核心方法论
2. **文化刻板印象**: 避免过度简化, 展现文化的复杂性
3. **时效性问题**: 企业文化会演变, 定期更新
4. **地域差异**: 不同地区办公室可能有文化差异
