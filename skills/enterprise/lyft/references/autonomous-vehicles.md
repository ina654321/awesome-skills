# Lyft Autonomous Vehicle Strategy

## Strategic Overview

### Philosophy
Lyft operates as a **network integrator**, not an AV developer. The company partners with autonomous vehicle companies to offer AV rides through the Lyft platform.

### Key Principle
> "We don't need to own the technology to benefit from it."

## Hybrid Fleet Strategy

### Service Tiers

```
┌─────────────────────────────────────────────────────┐
│                  Lyft Platform                       │
├──────────────┬──────────────┬───────────────────────┤
│  AV Standard │   AV Plus    │    Human Driver       │
│  (Base AV)   │  (Premium)   │    (All rides)        │
└──────────────┴──────────────┴───────────────────────┘
```

| Tier | Description | Pricing |
|------|-------------|---------|
| **AV Standard** | Driverless, basic experience | 10-20% discount |
| **AV Plus** | Driverless, premium features | Comparable to Lyft Lux |
| **Human Driver** | Traditional ride | Standard pricing |

### Technical Integration

| Component | Description |
|-----------|-------------|
| **AV Telemetry** | Real-time position, battery, ETA |
| **Fleet Orchestration** | Balance AV vs. human supply |
| **Dispatch Logic** | Rider choice + availability matching |
| **Safety Protocols** | Remote monitoring, emergency stop |

### Business Model

| Element | Approach |
|---------|----------|
| **Pricing** | AV rides at discount (no driver to pay) |
| **Driver Impact** | Retraining programs, other platform roles |
| **Peak Handling** | Human drivers for surge/unexpected demand |
| **Density Requirements** | AVs only where volume justifies deployment |

## Rider Experience

### First-Time AV Ride

1. **In-App Tutorial** - How to identify vehicle, unlock
2. **Pickup Zone** - Designated AV zones (more precise)
3. **Vehicle Identification** - Clear markings, app notification
4. **In-Ride Controls** - Touchscreen for climate, music
5. **Feedback Loop** - AV-specific rating categories

### UX Considerations

| Aspect | Design Choice |
|--------|---------------|
| **Communication** | Minimal (no driver small talk) |
| **Customization** | Pre-set preferences in app |
| **Assistance** | Remote support available |
| **Accessibility** | Designed for all abilities |

## Marketplace Considerations

### Driver Transition Strategy

| Initiative | Description |
|------------|-------------|
| **Retraining** | Other platform roles (support, ops) |
| **Hybrid Period** | Long transition with human drivers always available |
| **Earnings Protection** | Guaranteed minimums during transition |

### Pricing Strategy

| Scenario | Approach |
|----------|----------|
| **Base Case** | AV rides 10-20% cheaper |
| **Peak Times** | Human drivers handle surge |
| **Rural Areas** | Human drivers only (density too low) |

## Challenges & Considerations

| Challenge | Lyft Response |
|-----------|---------------|
| **Safety Concerns** | Opt-in only, clear communication |
| **Weather Limitations** | Human backup in bad conditions |
| **Regulatory Variance** | City-by-city compliance |
| **Public Acceptance** | Gradual rollout, education |
