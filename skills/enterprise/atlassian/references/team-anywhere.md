# Team Anywhere: Atlassian's Distributed Work Model

## Overview

Team Anywhere is Atlassian's default operating model — not a perk, but a fundamental way of working that enables global talent access and improved productivity.

## The Four Pillars

### 1. Async by Default
Replace meetings with written communication

**Tools:**
- Confluence for documentation and decisions
- Loom for video explanations
- Jira for work tracking
- Slack for quick questions

**Guidelines:**
- Write updates instead of status meetings
- Record demos with Loom instead of live sessions
- Document decisions in Confluence with context
- Use Jira comments for technical discussions

### 2. Open by Default
Make information broadly accessible

**Practices:**
- Public Slack channels for work topics
- Open Confluence spaces by default
- Shared metrics and dashboards
- Transparent financials in town halls
- Blameless post-mortems shared company-wide

### 3. Intentional Connection
Build relationships through purposeful gatherings

**Virtual Rituals:**
- Weekly Loom updates
- Monthly "Donut" pairings
- Quarterly virtual game sessions
- #watercooler Slack channel

**In-Person ITGs (Intentional Together Gatherings):**
- Frequency: Quarterly
- Duration: 3-4 days
- Activities: Strategy, workshops, team building
- Locations: Rotate between hubs

### 4. Timezone Awareness
Structure teams for effective collaboration

**Overlap Requirements:**
- Minimum 4 hours of overlap for collaboration
- Teams structured within 2-3 time zones max
- Handoff documentation for timezone transitions

**Example Team Topology:**
```
Sydney (UTC+10) → Bangalore (UTC+5:30) = 4.5h overlap
Bangalore (UTC+5:30) → Berlin (UTC+1) = 3.5h overlap
Berlin (UTC+1) → SF (UTC-8) = async only
```

## The 90-Day Rule

Employees can work from any supported country for up to 90 days/year without triggering tax or legal implications.

**Supported Countries (13+):**
- United States
- Australia
- United Kingdom
- Germany
- India
- Japan
- Netherlands
- Philippines
- Poland
- South Korea
- Turkey
- Vietnam
- And more

**Process:**
1. Notify manager
2. Confirm timezone compatibility
3. Update location in systems
4. Work normally

### Channel Usage

**Use Confluence For:**
- Decisions that need to persist
- Documentation and runbooks
- Meeting notes and agendas
- Project status and roadmaps

**Use Jira For:**
- Work tracking and assignments
- Technical discussions on issues
- Sprint planning and retrospectives
- Bug reports and feature requests

**Use Slack For:**
- Quick questions (< 5 min answer)
- Real-time coordination during overlap
- Social connection and team bonding
- Incident response coordination

**Use Loom For:**
- Demos and walkthroughs
- Complex explanations
- Weekly async standups
- Code review explanations

## Meeting Policy

### Required Synchronous
- Sprint Planning (2 hours, bi-weekly)
- Retrospective (1 hour, bi-weekly)
- 1:1s (30 min, weekly)
- Quarterly ITG (3-4 days)

### Async Replacements
- Daily Standup → Loom updates in #standups
- Code Review → Bitbucket with detailed comments
- Design Reviews → Confluence with async feedback
- Status Updates → Weekly Confluence page

### Meeting-Free Wednesdays
- Purpose: Deep work focus time
- Exception: P1 incidents only
- Communication: Async only

## Geo-Based Compensation

Atlassian uses location-based pay to ensure fairness across markets.

**Australia:**
- Two tiers: NSW/VIC vs other states

**United States:**
- Three zones based on cost-of-living

**Benefits:**
- Full benefits including healthcare and equity
- Monthly internet and mobile reimbursement
- Home office setup stipend
- Travel reimbursement for team gatherings
