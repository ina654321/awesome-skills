## Grocery Delivery

### Market Opportunity
- US grocery market: ~$800B annually
- Online penetration: ~10% (growing rapidly)
- Key competitors: Instacart, Amazon Fresh, Walmart+

### DoorDash Strategy
| Element | Approach |
|---------|----------|
| **Partnerships** | Safeway, Albertsons, ALDI, Sprouts, Meijer |
| **DashMart** | Dark stores for rapid delivery (30 min) |
| **DoubleDash** | Add grocery to restaurant order |
| **Model** | Marketplace (not holding inventory) |

### Operational Differences: Grocery vs Restaurant

| Dimension | Restaurant | Grocery | Implication |
|-----------|------------|---------|-------------|
| Order size | 1-4 items | 15-50 items | Longer shopping |
| Avg order value | $25-35 | $60-100 | Higher revenue/order |
| Prep time | 10-20 min | 30-60 min | Different Dasher model |
| Substitutions | Rare | Common (30%+) | Real-time communication |
| Temperature | Hot food | Frozen/cold/ambient | Multi-zone bags |
| Weight | <10 lbs | 20-50 lbs | Vehicle requirements |
| Scheduling | On-demand | Scheduled windows | Different demand |

### Grocery Logistics Model

```
┌─────────────────────────────────────────────────────────┐
│              GROCERY FULFILLMENT FLOW                   │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  1. Customer places order via app                       │
│     └── Select store, items, substitutions prefs        │
│                                                         │
│  2. Order transmitted to store                          │
│     └── Integration with store systems                  │
│                                                         │
│  3. Dasher assigned                                     │
│     └── Dedicated grocery Dashers (car required)        │
│                                                         │
│  4. Shopping                                            │
│     └── Dasher picks items from aisles                  │
│     └── Real-time substitution approval                 │
│                                                         │
│  5. Checkout                                            │
│     └── Store payment integration                       │
│                                                         │
│  6. Delivery                                            │
│     └── Temperature-controlled transport                │
│     └── Customer handoff                                │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### Key Metrics

| Metric | Target | Challenge |
|--------|--------|-----------|
| Fulfillment accuracy | >98% | Many items = higher error risk |
| Substitution rate | <15% | Inventory sync critical |
| Shopping time/item | <45 seconds | Dasher efficiency |
| Customer satisfaction | >4.5/5 | High expectations |
| Temperature compliance | 100% | Cold chain integrity |

## DashMart

### Concept
Dark stores (fulfillment centers) for rapid delivery of essentials, operated by DoorDash.

### Selection
- Groceries (produce, dairy, pantry)
- Household items
- Convenience goods
- Local specialties

### Operations
| Aspect | Approach |
|--------|----------|
| Locations | Urban centers |
| Delivery time | <30 minutes |
| Hours | Often 24/7 |
| Staffing | Employees (not Dashers) for picking |
| Delivery | Dasher pickup from DashMart |

### Competitive Position
- Vs. Instacart: Faster (30 min vs. 2 hours)
- Vs. Gopuff: Similar model, broader selection
- Vs. Amazon: Faster, local inventory

## DoubleDash

### Feature
Allow customers to add items from nearby stores to an active restaurant order.

### Use Case
"Ordering dinner + need ice cream from convenience store"

### Benefits
| Stakeholder | Benefit |
|-------------|---------|
| **Consumer** | One delivery fee, convenience |
| **Dasher** | Higher earnings per trip |
| **Platform** | Increased GOV, efficiency |
| **Merchant** | Incremental order |

### Implementation
- Geofenced nearby partners
- Timing coordination
- Route optimization
- Single delivery fee

## DoorDash Drive

### Concept
White-label delivery service for merchants to offer delivery on their own websites/apps.

### Target Merchants
- Large chains wanting brand control
- Grocers with existing e-commerce
- Retailers adding delivery
- Businesses without delivery infrastructure

### Value Proposition
| For Merchant | For DoorDash |
|--------------|--------------|
| Brand control | Incremental revenue |
| Customer data access | Utilize excess Dasher capacity |
| Custom experience | B2B relationships |
| No fleet investment | Higher margin business |

## Strategic Analysis

### Why New Verticals Matter

| Factor | Impact |
|--------|--------|
| **TAM Expansion** | Addressable market 3-4x restaurant delivery |
| **Frequency** | Grocery = weekly vs. food = occasional |
| **Retention** | More use cases = stickier platform |
| **Margins** | Grocery/retail often better economics |
| **Competition** | Differentiate from pure food competitors |

### Challenges

| Challenge | Mitigation |
|-----------|------------|
| **Complex logistics** | Dedicated grocery Dashers, training |
| **Substitution complexity** | ML recommendations, customer preferences |
| **Inventory sync** | Real-time POS integration |
| **Quality control** | Temperature monitoring, handling protocols |
| **Customer education** | Marketing, app experience |

### Growth Trajectory

```
2022: ~10% of GOV
2023: ~15% of GOV
2024: ~20% of GOV
2025E: ~25% of GOV
2030E: 35-40% of GOV
```
