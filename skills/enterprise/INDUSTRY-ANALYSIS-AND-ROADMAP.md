## 🎯 Top 1企业筛选标准

### 选择维度
1. **行业影响力**: 定义行业标准，被同行效仿
2. **方法论创新**: 有独特的管理和工作方法论
3. **人才口碑**: 吸引顶尖人才，培养行业领袖
4. **文化独特性**: 区别于同行的鲜明文化特征
5. **资料可及性**: 有足够的公开资料进行研究

### 排除标准
- 过于封闭/保密的企业 (如某些对冲基金)
- 方法论过于通用的企业
- 负面新闻过多的企业

## 📚 方法论提取框架

### 每个企业Skill需要提取的核心要素

#### 1. 决策框架 (Decision Framework)
```
示例: Amazon的14 Leadership Principles
- Customer Obsession
- Ownership
- Invent and Simplify
- ...
```

#### 2. 工作方法 (Working Methodology)
```
示例: Amazon的Working Backwards
1. Write the Press Release
2. Write FAQ
3. Define User Experience
4. Write User Manual
5. Then build
```

#### 3. 文化特质 (Cultural Traits)
```
示例: Netflix的Freedom & Responsibility
- High talent density enables high freedom
- Context not Control
- No vacation policy
- 4A Feedback
```

#### 4. 评估标准 (Evaluation Criteria)
```
示例: Netflix的Keeper Test
"If this person told me tomorrow they're leaving, 
how hard would I fight to keep them?"
```

#### 5. 反模式 (Anti-patterns)
```
示例: Amazon
❌ Customer Obsession ≠ Customer Servitude
❌ Bias for Action ≠ Recklessness
```

## 🎓 可复制的开发流程

### Step 1: 信息收集 (2-3天)
```bash
# 搜索模板
"{company} leadership principles"
"{company} culture deck"
"{company} interview questions"
"{company} engineering culture"
"working at {company}"

# 书籍搜索
"{company} way"
"{founder} biography"
"{company} case study"

# 视频/播客
YouTube: "{company} culture"
Podcasts: "{founder} interview"
```

### Step 2: 方法论提取 (1天)
1. 列出核心价值观/原则 (3-7条)
2. 识别独特工作方法 (2-4个)
3. 找出决策框架 (1-2个)
4. 总结面试考察点
5. 记录常见误解/反模式

### Step 3: Skill编写 (1-2天)
1. 基于TEMPLATE-enterprise.md
2. 填充16个标准章节
3. 重点编写:
   - §5 Professional Toolkit (方法论详情)
   - §8 Scenario Examples (3+场景)
   - §9 Anti-patterns (8个反模式)

### Step 4: 质量验证 (半天)
```bash
./scripts/skill-scorer.sh path/to/SKILL.md
```
- SDI ≥ 2.0
- 16 sections complete
- 4+ risks
- 3+ scenarios
- 8 anti-patterns

### Step 5: 发布 (半天)
1. 更新CATALOG.md
2. 更新本ROADMAP文档
3. 更新总Skill数量
4. 记录经验教训
