## § 5 · Workflow: Nintendo Game Development

### Phase 1: Discovery (4-8 weeks)

**Objective:** Find the "delicious moment" and validate hardware fit

**Activities:**
- [✓] Hardware dev kit deep dive (2+ weeks hands-on)
- [✓] Paper prototype core mechanics
- [✓] Kid playtest with physical prototype
- [✓] Document "magical moments" with unique hardware features
- [✗] No code written
- [✗] No concept art produced

**Deliverable:** Concept document with:
- Core mechanic description
- Hardware justification
- Target demographic
- Competitive differentiation

### Phase 2: Vertical Slice (6-12 months)

**Objective:** One complete, polished experience proving the concept

**Activities:**
- [✓] One complete level/scenario
- [✓] Core mechanic fully implemented
- [✓] Target framerate locked (typically 60fps)
- [✓] "Delicious moment" present and testable
- [✓] Hardware features integrated
- [✗] No full game progression
- [✗] Placeholder art acceptable

**Quality Gate:** Would Miyamoto say "delicious" when playing?

### Phase 3: Production (2-5 years)

**Objective:** Create full game from approved slice

**Activities:**
- [✓] Content creation following vertical slice template
- [✓] Weekly playtests with target demographics
- [✓] Localization-ready text implementation
- [✓] Bug fix buffer (3+ months minimum)
- [✗] No new mechanics added
- [✗] No feature creep permitted

**Milestones:**
- Alpha: All content in, playable start-to-finish
- Beta: Feature-complete, focus on polish
- Master: Lot check ready, no known critical bugs

### Phase 4: Post-Launch

**Nintendo Philosophy:**
- Minimal DLC—complete experience at launch
- No day-one patches (ship when ready)
- Long-tail sales focus (evergreen titles)
- Player feedback informs sequels, not patches
