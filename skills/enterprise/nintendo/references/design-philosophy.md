# Nintendo Design Philosophy

## Core Tenets

### 1. Gameplay First (玩法至上)

**Principle:** No story, graphics, or technology should compromise core mechanics.

**Miyamoto Test:** Can the game be fun with placeholder graphics and no narrative?

**Application:**
- Prototype mechanics with cubes/spheres before any art
- If it's not fun in greybox, it's not the art's fault
- Core loop must be satisfying immediately

**Examples:**
- Super Mario 64: Spent months just running and jumping in empty world
- Breath of the Wild: Miyamoto climbed trees for hours in prototype
- Splatoon: Ink mechanic tested before any squid theming

### 2. Hardware-Software Synergy

**Principle:** Create experiences that could only exist on Nintendo hardware.

**Test:** Could this experience exist on competing hardware?

**Historical Leverage:**
| Hardware | Unique Feature | Enabled Experience |
|----------|----------------|-------------------|
| DS | Touch screen | Brain Age, Trauma Center |
| DS | Dual screens | Map on bottom, gameplay on top |
| Wii | Motion controls | Wii Sports, Wii Fit |
| Wii U | GamePad | Asymmetric multiplayer |
| Switch | Hybrid portable/home | Zelda anywhere, Mario Kart anytime |
| Switch 2 | Mouse Joy-Con | PC-like precision, new genres |

**Application:**
- Spend weeks with dev kit before design docs
- Discard concepts that could work elsewhere
- Build around "magical moments" with unique inputs

### 3. Subtractive Design

**Principle:** Start with 20 ideas, reduce to 1 core mechanic. Add only when it multiplies the core.

**Process:**
1. Brainstorm extensively
2. Find the "delicious moment"
3. Remove everything that doesn't support it
4. Add complexity only when it deepens the core

**Example - Mario Odyssey:**
- Core: Capture mechanic (Cappy possession)
- Deep: Momentum transfer, puzzle solutions, exploration
- Simple: One button throw
- Depth: Advanced techniques emerge naturally

**Example - Pikmin:**
- Core: Command small creatures
- Actions: Throw, whistle, dismiss (only 3)
- Depth: AI behaviors, multitasking, strategy

### 4. Universal Accessibility

**Principle:** Games playable by ages 5 to 95.

**Tests:**
- Can a 5-year-old enjoy the first 10 minutes?
- Can a grandparent understand the goal?
- Can someone who's never played games succeed?

**Implementation:**
- Intuitive controls (motion, touch, simple buttons)
- Visual clarity over photorealism
- Difficulty through depth, not complexity
- Super Guide for struggling players

**Kid-Testing Protocol:**
1. Weekly playtests with actual children
2. Observe without guiding
3. Note exact moment they stop having fun
4. If stuck, design is wrong—not the player

### 5. The 30-Second Rule

**Principle:** Player must understand the fun within 30 seconds.

**Guidelines:**
- No tutorials longer than 3 steps
- Teach through level design, not text
- First level teaches everything without explicit instruction

**Example - Super Mario Bros. 1-1:**
1. Goomba approaches: Player learns to jump
2. Question block: Player learns to hit blocks
3. Power-up appears: Player learns mushrooms help
4. Pipe: Player learns verticality
5. Gap: Player learns jumping is essential
6. All without a single text prompt

### 6. When It's Ready

**Principle:** Ship only when "surprising and delighting" is achieved.

**Miyamoto Quote:** "A delayed game is eventually good, but a rushed game is forever bad."

**Examples:**
- The Legend of Zelda: Breath of the Wild: Delayed 2 years
- Animal Crossing: New Horizons: Delayed to ensure quality
- Metroid Prime 4: Restarted development with Retro Studios

**Financial Philosophy:**
- Conservative cash reserves ($12B+) enable patience
- No quarterly pressure compromising quality
- Long-term IP value over short-term sales

### 7. Multiplicative Gameplay

**Principle:** Systems should interact to create emergent experiences.

**Example - Breath of the Wild:**
- Fire + grass = spread
- Metal + lightning = danger
- Ice + water = frozen platform
- Any combination works as expected

**Result:** Players discover solutions developers didn't explicitly design.

### 8. Environmental Storytelling

**Principle:** Tell story through world design, not cutscenes.

**Techniques:**
- Visual world-building (ruins suggest history)
- Enemy placement tells story
- Item descriptions add lore
- Minimal dialogue, maximum implication

**Example - Metroid Prime:**
- Scan logs reveal Chozo civilization
- Environment shows their fall
- No cutscenes required for emotional impact

### 9. Polish as Priority

**Principle:** Frictionless feel through endless iteration.

**Examples:**
- Mario jump arc: Tweaked for perfect feel across every game
- Zelda sword swings: Frame-perfect response
- Splatoon ink: Months refining movement physics

**Miyamoto's "Delicious":**
The satisfying moment when controls, feedback, and result align perfectly.

### 10. Innovation Over Iteration

**Principle:** Create new genres rather than perfect existing ones.

**Historical Innovations:**
| Year | Innovation | Game |
|------|------------|------|
| 1985 | Side-scrolling platformer | Super Mario Bros. |
| 1986 | Action-adventure template | The Legend of Zelda |
| 1996 | 3D platforming | Super Mario 64 |
| 1998 | Lock-on combat | Ocarina of Time |
| 2004 | Touch screen gaming | DS launch titles |
| 2006 | Motion control | Wii Sports |
| 2017 | Open-air physics | Breath of the Wild |
| 2020 | Ring controller | Ring Fit Adventure |

## Contrarian Philosophy

### Against Industry Trends

| Industry Trend | Nintendo Response |
|----------------|-------------------|
| Photorealistic graphics | Stylized art, readability first |
| Cinematic storytelling | Environmental narrative |
| Annual sequels | 4-6 year development cycles |
| Online-only multiplayer | Local multiplayer priority |
| Free-to-play mechanics | Full experience at purchase |
| Day-one patches | Ship when ready |
| Focus group design | Designer intuition + kid testing |
| AI-generated content | Human-crafted experiences |

**Miyamoto on AI (2024):**
> "It might seem like we are just going the opposite direction for the sake of going in the opposite direction, but it really is trying to find what makes Nintendo special."

### Yamauchi's Wisdom

> "We are not good at fighting. We are weak—so don't go picking fights with other companies."

**Interpretation:** Nintendo avoids direct competition (specs, graphics wars) and creates new markets instead.

## Development Culture

### Kyoto Style (EPD Headquarters)
- Traditional hierarchy
- Miyamoto mentoring
- Strict quality standards
- Flagship IP focus

### Tokyo Style (EPD Tokyo)
- Faster iteration
- Younger teams
- Online/multiplayer focus
- New IP experiments (Splatoon)

### Common Elements
- Designer authority (not producer-driven)
- Long development timelines
- In-house engine/tools
- Low turnover (avg 14.4 years in Japan)

## Quality Checklist

Before any Nintendo game ships, it must pass:

- [ ] Core mechanic is "delicious"
- [ ] 5-year-old can play first 10 minutes
- [ ] Hardware features utilized
- [ ] No tutorial longer than 3 steps
- [ ] Super Guide ready (if applicable)
- [ ] 60fps locked (action games)
- [ ] Playtested by non-gamers
- [ ] Miyamoto/director sign-off
- [ ] Lot check passed (technical certification)
- [ ] Would players love this in 10 years?
