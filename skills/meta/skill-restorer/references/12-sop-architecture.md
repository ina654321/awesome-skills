# SOP 3: Architecture Design

## Purpose

Plan optimal skill structure before writing content.

## Input

Diagnosis Report + Research Notes

## Output

Architecture Blueprint

## Process

### Step 1: Design System Prompt (10 min)

**§1.1 Identity & Worldview**:
```
Your Professional DNA:
├── [Role 1]: [Description]
├── [Role 2]: [Description]
└── [Role 3]: [Description]

Your Context:
├── [Company/Domain overview]
├── [Key data points]
└── [Success metrics]
```

**§1.2 Decision Framework**:
```
The [Name] Hierarchy:
1. [Priority 1]
2. [Priority 2]
3. [Priority 3]

Quality Gates:
├── Gate 1: [Criteria]
├── Gate 2: [Criteria]
└── Gate 3: [Criteria]
```

**§1.3 Thinking Patterns** (minimum 3):
```
Pattern 1: [Name]
├── Principle: 
├── Application: 
└── Example: 
```

### Step 2: Map Domain Knowledge (5 min)

Identify what domain knowledge to include.

```
Domain Knowledge Sections:
├── Company/Organization Data
├── Methodology Details
├── Industry Context
├── Key Terminology
└── Benchmarks and Metrics
```

### Step 3: Design Workflow (5 min)

Plan 4-5 phases with Done/Fail criteria.

```
Phase 1: [Name]
├── Objective: 
├── Activities: 
├── Done Criteria:
│   ├── [ ]
│   └── [ ]
└── Fail Criteria:
    ├── [ ]
    └── [ ]
```

### Step 4: Plan Examples (5 min)

Design 5 detailed scenarios.

```
Example 1: [Type - Standard]
├── Context: 
├── User Request: 
└── Expected Response: 

Example 2: [Type - Complex]
...

Example 3: [Type - Edge Case]
...

Example 4: [Type - Error]
...

Example 5: [Type - Ethics]
...
```
