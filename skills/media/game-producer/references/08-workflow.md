# Standard Workflow

## 8.1 Pre-Production

```
Pre-Production Workflow
├── 1. Concept Development
│   ├── Initial pitch and vision document
│   ├── Market analysis and competitive review
│   ├── Target audience definition
│   └── High-level budget estimate
│
├── 2. Greenlight
│   ├── Pitch presentation to stakeholders
│   ├── Budget and timeline approved
│   ├── Team headcount defined
│   └── Publisher/investor secured
│
├── 3. Game Design Document
│   ├── Core gameplay loop defined
│   ├── Feature list prioritized
│   ├── Narrative outline complete
│   ├── Monetization model documented
│   └── Technical requirements outlined
│
├── 4. Vertical Slice
│   ├── Core 10% of game playable
│   ├── Art style locked
│   ├── Core mechanics prototyped
│   └── Iteration with team feedback
│
└── 5. Production Ready
    ├── Greenlight confirmed
    ├── Full team hired/contracted
    ├── Detailed schedule created
    └── Pipeline tools established
```

## 8.2 Production Management

```
Production Workflow
├── Weekly Sprint Cycle
│   ├── Sprint planning (Monday)
│   ├── Daily standups (daily, 15 min)
│   ├── Playable builds (mid-week)
│   ├── Bug bash (end of week)
│   └── Sprint review/demo (Friday)
│
├── Milestone Tracking
│   ├── Burn-down charts updated daily
│   ├── Risk register reviewed weekly
│   ├── Scope managed carefully
│   └── Change control process active
│
└── Communication
    ├── Weekly status to publisher/studio head
    ├── Monthly all-hands
    ├── Quarterly stakeholder review
    └── Real-time issue escalation
```

## 8.3 Quality Assurance Workflow

```
QA Process
├── Internal Testing
│   ├── Playtests (weekly minimum)
│   ├── Feature verification
│   ├── Regression testing
│   └── Balance tuning
│
├── External Testing
│   ├── Focus groups (quarterly)
│   ├── Beta testing (public/closed)
│   ├── Localization testing
│   └── Platform certification testing
│
└── Bug Management
    ├── Severity 1 (Crash): 24-hour fix
    ├── Severity 2 (Major): 1-week fix
    ├── Severity 3 (Minor): Milestone target
    └── Severity 4 (Polish): As time allows
```

## 8.4 Launch Checklist

```
Pre-Launch (60 days)
├── Build Finalization
│   ├── Content locked
│   ├── All bugs severity 1-2 resolved
│   ├── Performance optimized
│   └── Platform submission builds created
│
├── Marketing
│   ├── Review copies sent
│   ├── Press kits distributed
│   ├── Trailer and screenshots finalized
│   └── Launch day events planned
│
├── Operations
│   ├── Live ops team staffed
│   ├── Server capacity planned
│   ├── Customer support trained
│   └── Community guidelines ready
│
└── Platform
    ├── Certification submitted
    ├── Age ratings confirmed
    ├── Regional release dates coordinated
    └── Day-one patch ready
```

## 8.5 Post-Launch Support

| Activity | Timeline | Team |
|---------|----------|------|
| Day-one patch | 0-24 hours | Dev + ops |
| Hotfix | 1-7 days | Dev on-call |
| First DLC | 1-3 months | Expansion team |
| Major patch | 3-6 months | Roadmap-driven |
| Sequel planning | 6-12 months | Studio decision |

## 8.6 Crisis Management

| Issue | Response | Team Lead |
|-------|---------|-----------|
| Critical bug discovered | Immediate all-hands | Tech lead |
| Server outage | Ops escalation | DevOps |
| Community backlash | Community + PR | Community manager |
| Platform rejection | Emergency fix | Tech lead |
| Security vulnerability | Disclosure + patch | Security lead |
