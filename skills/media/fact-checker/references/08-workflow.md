# Standard Workflow

## 8.1 Claim Intake

```
Claim Evaluation Process
├── 1. Claim Received
│   ├── Document exact wording of claim
│   ├── Note source and platform
│   ├── Identify key factual elements to check
│   └── Assess news value (why check this?)
│
├── 2. Initial Assessment
│   ├── Is the claim checkable? (factual, not opinion)
│   ├── Is it significant? (public interest)
│   ├── Is it recent enough? (avoid stale claims)
│   └── Can I verify it? (access to sources)
│
├── 3. Scope Definition
│   ├── Narrow to specific factual claims
│   ├── Break complex claims into checkable parts
│   ├── Identify most impactful elements
│   └── Set deadline (usually 24-72 hours)
│
└── 4. Assignment Decision
    ├── Assign to fact-checker
    ├── Assign to senior reviewer
    └── Decline if not appropriate
```

| Step | ✓ Done | ✗ Fail |
|------|--------|--------|
| Claim Received | Exact wording documented, source noted | Vague recollection, source unknown |
| Initial Assessment | All 4 questions answered clearly | Unclear if checkable/significant |
| Scope Definition | Specific claims defined, deadline set | Scope too vague or too broad |
| Assignment | Decision made and communicated | No clear owner |

## 8.2 Fact-Check Process

```
Fact-Check Workflow
├── Step 1: Claim Disassembly
│   ├── Break into individual factual claims
│   ├── Identify dates, names, numbers, locations
│   ├── Remove opinion elements
│   └── Note context and framing
│
├── Step 2: Research
│   ├── Primary sources first (official data, studies)
│   ├── Secondary sources (established news)
│   ├── Expert consultation (if needed)
│   └── Direct verification (documents, records)
│
├── Step 3: Verification
│   ├── For each factual claim
│   ├── Find supporting or contradicting evidence
│   ├── Assess quality of evidence
│   └── Note expert consensus
│
├── Step 4: Rating Decision
│   ├── Based on evidence weight
│   ├── Consider context and framing
│   ├── Assess intent and impact
│   └── Consult senior reviewer
│
└── Step 5: Write-up
    ├── Clear explanation of methodology
    ├── Summary verdict first
    ├── Evidence presented fairly
    ├── Links to sources
    └── Context about why it matters
```

| Step | ✓ Done | ✗ Fail |
|------|--------|--------|
| Claim Disassembly | All factual elements isolated, opinions separated | Mixed fact/opinion, elements missed |
| Research | 3+ independent sources found, primary preferred | Single source only, no primary sources |
| Verification | Each element has supporting/refuting evidence | Unsupported claims remain |
| Rating Decision | Rating matches evidence weight, senior consulted | Rating disconnected from evidence |
| Write-up | Verdict first, methodology clear, sources linked | No clear verdict, sources missing |

## 8.3 Expert Consultation

| When to Use | How to Select | What to Ask |
|------------|--------------|-------------|
| Scientific claims | Published in field, no conflicts | "What's the consensus?" |
| Historical claims | Published historians | "What does evidence show?" |
| Technical claims | Industry professionals | "Is this accurate?" |
| Statistical claims | Statisticians, researchers | "Is this interpretation valid?" |
| Legal claims | Attorneys, legal scholars | "Is this accurate statement of law?" |

### Expert Vetting Checklist

- Published in peer-reviewed journals or respected publications
- Current or recent work in relevant field
- No financial conflict of interest
- No ideological/political organization affiliation
- Not quoted out of context

## 8.4 Viral Hoax Verification

```
Viral Claim Verification Steps
├── 1. Search the Claim
│   ├── Quote search in news databases
│   ├── Check Snopes, PolitiFact, FactCheck.org
│   ├── Google reverse image search
│   └── WHOIS lookup for suspicious sites
│
├── 2. Source Verification
│   ├── Is there an original source?
│   ├── Is the source credible?
│   ├── Has the source confirmed?
│   └── Is context missing or distorted?
│
├── 3. Claim Analysis
│   ├── What is actually being said?
│   ├── Is it verifiable factual claim?
│   ├── What evidence exists?
│   └── Is it satire/ parody?
│
└── 4. Documentation
    ├── Screenshots of original claim
    ├── Evidence collected
    ├── Sources documented
    └── Timeline of verification
```

| Step | ✓ Done | ✗ Fail |
|------|--------|--------|
| Search the Claim | All debunker sites checked, reverse search done | Only one source checked |
| Source Verification | Original source found and verified | No original source, unverified claims |
| Claim Analysis | Claim classified (factual/satire/misleading) | Unclear what type of claim |
| Documentation | Screenshots saved, evidence organized | No proof of original claim |

## 8.5 Correction Process

```
When Fact-Check is Wrong
├── Identify the Error
│   ├── Internal discovery or reader feedback
│   ├── Assess severity
│   └── Document original claim and verdict
│
├── Notify Editor Immediately
│   ├── Don't wait or hide the error
│   ├── Correct verdict and reasoning
│   └── New evidence reviewed by senior
│
├── Publish Correction
│   ├── Transparent correction notice
│   ├── Update digital article
│   ├── Social media correction
│   └── Note when/how error occurred
│
└── Post-Correction Review
    ├── What failed in verification?
    ├── Process improvements needed
    └── Lessons for team
```

| Step | ✓ Done | ✗ Fail |
|------|--------|--------|
| Identify Error | Severity assessed, original documented | Error ignored or minimized |
| Notify Editor | Editor informed within 24 hours | Delayed notification |
| Publish Correction | All platforms updated, transparent notice | Incomplete update, misleading correction |
| Post-Correction Review | Root cause identified, process improved | Same error can recur |
