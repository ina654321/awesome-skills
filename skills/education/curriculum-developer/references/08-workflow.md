## 8. Standard Workflow

### 8.1 Curriculum Development Process

```
Phase 1: Analysis (20% of time)
├── Needs analysis: What must learners be able to do?
├── Audience analysis: Who are they? What do they know?
├── Content analysis: What topics must be covered?
├── Constraint analysis: What are the limits?
└── [✓ Done]: Analysis document completed

Phase 2: Design (30% of time)
├── Write learning objectives (Bloom's verbs)
├── Design assessments (formative + summative)
├── Sequence learning experiences
├── Select instructional strategies
└── [✓ Done]: Design document approved

Phase 3: Development (35% of time)
├── Create content and materials
├── Develop assessments
├── Build activities and exercises
├── Apply UDL principles throughout
└── [✓ Done]: Pilot-ready curriculum

Phase 4: Implementation (10% of time)
├── Train instructors/facilitators
├── Pilot with sample learners
├── Gather feedback
└── [✓ Done]: Implementation plan ready

Phase 5: Evaluation (5% of time)
├── Analyze pilot data
├── Identify improvements
├── Document lessons learned
└── [✓ Done]: Revision plan created
```

### 8.2 Learning Objective Writing

```
Step 1: Identify the Domain
  → Cognitive (knowledge)
  → Affective (attitudes)
  → Psychomotor (skills)

Step 2: Select the Level
  → Bloom's: Remember → Understand → Apply → Analyze → Evaluate → Create
  → Anderson: Remember → Understand → Apply → Analyze → Evaluate → Create

Step 3: Choose Action Verbs
  → Avoid: know, understand, learn, appreciate
  → Use: define, list, compare, analyze, create, evaluate

Step 4: Write the Objective
  → [Audience] will be able to [action verb] [content] [condition/criteria]
  → Example: "Students will be able to analyze primary source documents 
              to identify bias with 80% accuracy"

[✓ Done]: Specific, measurable, achievable, relevant, time-bound objectives
```
