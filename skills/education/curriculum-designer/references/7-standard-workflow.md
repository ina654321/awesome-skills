## § 7 · Standard Workflow

### Phase 1: Analysis and Planning

| Step | Activity | Done Criteria | Fail Criteria |
|------|----------|---------------|---------------|
| 1 | Conduct needs assessment and context analysis | Stakeholder input documented; constraints identified | Designing without understanding context |
| 2 | Review standards and develop competency framework | Competencies defined with proficiency levels | Competencies too vague to assess |
| 3 | Create assessment blueprint | Every competency has aligned assessment method | Assessments designed after instruction |
| 4 | Map vertical and horizontal alignment | Scope-and-sequence draft completed | Isolated course design without system view |
| 5 | Develop implementation plan | Timeline, resources, PD plan documented | No plan for supporting teachers |

### Phase 2: Development and Review

| Step | Activity | Done Criteria | Fail Criteria |
|------|----------|---------------|---------------|
| 1 | Write unit plans with UbD template | All units have Stage 1, 2, 3 complete | Units without clear outcomes |
| 2 | Develop performance assessments | Rubrics created; exemplars collected | Assessments without criteria |
| 3 | Curate and develop instructional resources | Materials aligned and accessible | Resources collected without alignment check |
| 4 | Conduct alignment review | Independent review confirms standards alignment | No external review of alignment |
| 5 | Pilot test with feedback collection | Pilot data collected and analyzed | Full rollout without pilot |
