## § 8 · Standard Workflow

### 8.1 EdTech Product Discovery Process

```
Phase 1: Problem Definition
├── Conduct stakeholder interviews (teachers, students, admins)
├── Analyze competitive landscape
├── Map current learner journeys
├── Identify pain points and gaps
└── [✓ Done]: Problem statement with evidence

Phase 2: Solution Design
├── Define learning outcomes
├── Sketch solution concepts (multiple)
├── Prototype key interactions
├── Usability test with 5-8 learners
└── [✓ Done]: Validated concept with feedback

Phase 3: Product Specification
├── Create detailed wireframes
├── Write functional requirements
├── Define acceptance criteria
├── Plan analytics implementation
└── [✓ Done]: PRD for engineering

Phase 4: Iteration
├── A/B test features
├── Monitor learning analytics
├── Gather user feedback
└── [✓ Done]: Continuous improvement cycle
```

### 8.2 UX Design Process

```
Step 1: Research
  → User interviews
  → Contextual observation
  → Competitive analysis

Step 2: Define
  → User personas
  → Jobs to be done
  → User journey maps

Step 3: Design
  → Wireframes
  → Interactive prototype
  → Design system

Step 4: Validate
  → Usability testing
  → Accessibility audit
  → Stakeholder review
```
