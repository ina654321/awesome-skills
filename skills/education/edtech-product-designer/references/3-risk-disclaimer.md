## § 3 · Risk Disclaimer

| Risk | Severity | Description | Mitigation |
|------------|-----------------|-------------------|---------------------|
| **Pedagogy-Technology Mismatch** | 🔴 High | Adding tech features that don't improve learning wastes development time and confuses users | Start with learning objectives; verify each feature serves pedagogy |
| **Engagement-Vanity Metrics** | 🔴 High | High time-on-task or click counts don't equal learning; focus on outcome metrics | Measure actual learning gains, not just activity |
| **Accessibility Afterthought** | 🔴 High | Retrofitting WCAG compliance costs 10× more; plan upfront | Include accessibility in requirements from day one |
| **Feature Bloat** | 🟡 Medium | Every "nice to have" feature increases cognitive load and development cost | Apply RICE scoring; ruthlessly prioritize |
| **Privacy Compliance** | 🟡 Medium | COPPA, FERPA, GDPR apply to edtech; violations carry legal risk | Design data collection for consent; minimize data |

**⚠️ IMPORTANT**:
- This skill provides design guidance. Actual implementation requires compliance with applicable privacy laws (COPPA for children, FERPA for schools, GDPR for EU).
- Learning outcome claims (e.g., "improves test scores by 30%") require rigorous research methodology to substantiate.
- Accessibility is a legal requirement in many jurisdictions; WCAG 2.1 AA is the minimum standard.
