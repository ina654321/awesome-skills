# Standard Workflow

## 8.1 EdTech Product Design Process

```
Phase 1: Discovery & Research (2-4 weeks)
├── User interviews with teachers, students, administrators
├── Competitive analysis of 5-8 existing products
├── Define primary and secondary user personas
├── Identify unmet needs and opportunity areas
└── Regulatory landscape review (FERPA, GDPR, COPPA)

Phase 2: Define & Scope (2-3 weeks)
├── Create problem statement (Jobs-to-be-Done framework)
├── Define success metrics (DAU/MAU, completion rate, NPS)
├── Create MVP feature list (max 5 core features)
└── Stakeholder alignment and sign-off

Phase 3: Design (4-8 weeks)
├── Low-fidelity wireframes (paper or Figma)
├── User flow mapping
├── Mid-fidelity prototypes with interactions
├── Usability testing with 5-8 representative users
└── Iterate based on feedback

Phase 4: Build & Test (8-16 weeks)
├── Agile sprints with 2-week cycles
├── Continuous internal QA
├── Beta testing with pilot schools/classrooms
└── Accessibility audit (WCAG 2.1 AA)

Phase 5: Launch & Iterate
├── Soft launch (1-3 schools)
├── Collect usage data and qualitative feedback
├── Prioritize bug fixes and feature requests
└── Plan for broader rollout
```

## 8.2 User Research Workflow

1. **Recruit**: Minimum 5 participants per user type (teacher, student, admin)
2. **Screen**: Confirm participants match target persona criteria
3. **Interview guide**: Open-ended questions first, specific tasks second
4. **Session**: 45-60 minutes; observe without leading
5. **Analysis**: Affinity mapping; identify themes across sessions
6. **Synthesis**: Create actionable insights document for product team

## 8.3 Accessibility Workflow

1. Define scope: Which screens/features are in scope?
2. Automated scan: Use axe-core or Lighthouse
3. Manual keyboard navigation test
4. Screen reader test (NVDA, VoiceOver)
5. Color contrast check (minimum 4.5:1 for text)
6. Document findings in accessibility tracker
7. Prioritize and assign remediation
