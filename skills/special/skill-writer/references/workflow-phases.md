# Workflow Phases

## Phase Overview

| Phase | Objective | Duration | Done Criteria | Fail Criteria |
|-------|-----------|----------|---------------|---------------|
| 1. Assessment | Determine needs & tier | 5 min | Tier selected, path defined | Scope unclear |
| 2. Architecture | Design structure | 15 min | §1.1/1.2/1.3 blueprint | Missing sections |
| 3. Content | Write all sections | 60-120 min | All content filled | Generic content remains |
| 4. Examples | Create 5 scenarios | 30 min | 5 detailed examples | Examples shallow |
| 5. Validation | Verify quality | 15-30 min | Score ≥ 9.0 | Score < 8.5 |
| 6. Delivery | Package skill | 10 min | All files ready | Missing deliverables |

## Phase 1: Assessment

**Objective**: Understand user needs and recommend optimal path.

**Activities**:
1. Detect user's experience level
2. Determine skill scope
3. Select appropriate tier
4. Recommend entry point

**Done Criteria**:
- [ ] User experience level assessed
- [ ] Skill scope clearly defined
- [ ] Tier (Lite/Standard/Enterprise) selected
- [ ] Entry point (beginner/quick/standard/expert) recommended

**Fail Criteria**:
- [ ] Scope remains ambiguous
- [ ] Wrong tier selected for complexity
- [ ] User needs not understood

## Phase 2: Architecture

**Objective**: Design skill structure before writing.

**Activities**:
1. Design System Prompt (§1.1/1.2/1.3)
2. Map domain knowledge sections
3. Plan workflow phases
4. Design 5 examples
5. Plan progressive disclosure split

**Done Criteria**:
- [ ] System Prompt structure complete
- [ ] Domain knowledge mapped
- [ ] 5 examples designed
- [ ] Progressive disclosure plan ready

**Fail Criteria**:
- [ ] Missing §1.1/1.2/1.3 sections
- [ ] No example plan
- [ ] Flat structure (no progressive disclosure)

## Phase 3: Content

**Objective**: Write all skill sections.

**Activities**:
1. Write System Prompt
2. Fill domain knowledge
3. Create workflow
4. Add anti-patterns

**Done Criteria**:
- [ ] System Prompt complete (§1.1/1.2/1.3)
- [ ] Domain knowledge specific (no generics)
- [ ] Workflow has Done/Fail criteria
- [ ] Anti-patterns documented

**Fail Criteria**:
- [ ] Generic content remains
- [ ] Missing System Prompt sections
- [ ] No Done/Fail criteria

## Phase 4: Examples

**Objective**: Create 5 detailed scenarios.

**Activities**:
1. Design realistic contexts
2. Write user inputs
3. Create expert responses
4. Add concrete outputs

**Done Criteria**:
- [ ] 5 scenarios created
- [ ] Each has realistic context
- [ ] Specific data included
- [ ] Edge cases covered

**Fail Criteria**:
- [ ] < 5 examples
- [ ] Generic scenarios
- [ ] No specific data

## Phase 5: Validation

**Objective**: Verify EXEMPLARY quality.

**Activities**:
1. Run self-check
2. Check generic terms
3. Verify line counts
4. Simulate evaluator

**Done Criteria**:
- [ ] All dimensions checked
- [ ] Score ≥ 9.0
- [ ] Variance < 1.0
- [ ] All thresholds met

**Fail Criteria**:
- [ ] Score < 8.5
- [ ] Missing sections
- [ ] Generic content detected

## Phase 6: Delivery

**Objective**: Package and deliver skill.

**Activities**:
1. Create EVALUATION_REPORT.md
2. Verify file structure
3. Backup original
4. Final handoff

**Done Criteria**:
- [ ] All files created
- [ ] Backup preserved
- [ ] Report saved
- [ ] Structure verified

**Fail Criteria**:
- [ ] Missing files
- [ ] No backup
- [ ] No evaluation report
