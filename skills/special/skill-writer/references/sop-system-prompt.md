# SOP 2: System Prompt Design

## Purpose

Create §1.1, §1.2, and §1.3 sections that establish clear identity and decision-making framework.

## §1.1 Identity & Worldview

**Required Elements**:

1. **Role Title**: Specific, not generic
   - ✅ "You are a Strategic Planning Consultant"
   - ❌ "You are an expert assistant"

2. **Professional DNA**: 3-4 archetype descriptors
   ```
   - **Pattern Matcher**: Recognizes opportunity frameworks
   - **Data Synthesizer**: Combines market + internal data
   - **Decision Architect**: Structures complex choices
   - **Risk Calculator**: Quantifies uncertainty
   ```

3. **Context**: Your operating environment
   - What specification you follow
   - What tools you use
   - What success looks like

4. **Success Metrics**: Measurable outcomes
   ```
   - Strategy acceptance rate: >80%
   - Implementation success: >70%
   - Stakeholder alignment: >85%
   ```

## §1.2 Decision Framework

**Required Elements**:

1. **Priority Hierarchy**: Ordered list of what matters most
2. **Quality Gates**: Pass/fail checkpoints
3. **Trade-off Rules**: How to balance competing priorities

## §1.3 Thinking Patterns

**Required Elements**:

Minimum 3 patterns, each with:
- Pattern name
- When to use
- Concrete structure/checklist

## Verification Checklist

- [ ] §1.1 has specific role title (not "expert")
- [ ] §1.1 has 3-4 professional DNA descriptors
- [ ] §1.1 has specific success metrics
- [ ] §1.2 has priority hierarchy
- [ ] §1.2 has quality gates table
- [ ] §1.3 has 3+ thinking patterns
- [ ] Each pattern has concrete structure
- [ ] No generic terms anywhere
