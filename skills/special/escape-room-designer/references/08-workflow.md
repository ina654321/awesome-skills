# Standard Workflow

## 8.1 Design Phase

### Step 1: Theme Development
- Choose core narrative (mystery, horror, adventure, sci-fi, historical)
- Define setting and time period
- Identify emotional arc: tension → revelation → climax → resolution
- Create backstory: What happened here? What's at stake?

### Step 2: Story Integration
- Map narrative beats to room sections
- Identify what players learn in each area
- Connect story reveals to puzzle solutions
- Ensure every puzzle contributes to story understanding

### Step 3: Puzzle Sequence Design
- Create puzzle chain: How does one puzzle lead to the next?
- Design parallel puzzles: Multiple puzzles solvable simultaneously
- Plan optional puzzles: Not required for completion but add depth
- Build in reset capability: Puzzle can return to solved state

## 8.2 Construction Phase

### Step 4: Environment Design
- Choose set pieces and their functions
- Design lighting plan: ambient, spotlight, blacklight, strobe
- Plan sound design: background atmosphere, triggered audio
- Select materials: durability, safety, theme authenticity

### Step 5: Puzzle Implementation
- Build or source locks: mechanical, electronic, magnetic
- Create clue mechanisms: visible, UV, hidden, tactile
- Design custom props: puzzles, containers, set dressing
- Install automation: sensors, controllers, solenoids

### Step 6: Flow Optimization
- Design player movement patterns: circular, linear, branching
- Plan hint system: where hints appear, how they're triggered
- Create staging areas: where players gather, where they split
- Balance difficulty: time per puzzle, total room duration

## 8.3 Testing Phase

### Step 7: Internal Testing
- Walk through each puzzle solution multiple times
- Verify all locks work correctly
- Time each puzzle and full run
- Test all automation and effects

### Step 8: External Testing
- Recruit 5-10 playtesters from target audience
- Observe without intervening (or minimal intervention)
- Collect timing data: where they get stuck, where they rush
- Gather feedback: emotional response, clarity, engagement

### Step 9: Iteration & Refinement
- Adjust puzzle difficulty based on test data
- Improve ambiguous clues
- Fix flow bottlenecks
- Enhance moments that work well

## 8.4 Launch Phase

### Step 10: Documentation
- Write hint guides for staff
- Create troubleshooting procedures for common issues
- Document puzzle reset procedures
- Train operators on room mechanics

### Step 11: Soft Launch
- Run limited public sessions with close monitoring
- Collect post-session feedback
- Make final adjustments
- Full public launch
