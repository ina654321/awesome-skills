# Standards & Reference

## 7.1 Puzzle Categories & Mechanics

### Core Puzzle Types
| Type | Description | Example |
|------|-------------|---------|
| **Lock & Key** | Find key to open lock | Physical key, digital code, symbol match |
| **Sequence** | Arrange elements in correct order | Number lock, alphabetizing, timeline |
| **Pattern Recognition** | Identify and repeat visual patterns | Symbol grids, color sequences, shapes |
| **Physical/Manual** | Manipulate objects or environment | Hidden compartments, sliding panels, balancing |
| **Math/Logic** | Solve logical or numerical puzzles | Cipher codes, calculations, logic grids |
| **Search & Find** | Locate hidden elements in environment | UV light reveals, invisible ink, hidden objects |
| **Assembly** | Combine pieces to create solution | Puzzle pieces, fragment reassembly |
| **Meta-Puzzle** | Solution requires combining info from multiple puzzles | Final code requires solving 3 sub-puzzles |

### Difficulty Calibration
- **Easy**: 1-step solution, obvious clues, 60+ seconds
- **Medium**: 2-3 steps, some clues, 2-4 minutes
- **Hard**: Complex multi-step, minimal clues, 5+ minutes
- **Expert**: Meta-puzzles, requires collaboration, full team effort

## 7.3 Playtesting Standards

### Testing Protocol
1. **Alpha Testing**: Design team plays to verify puzzle logic works
2. **Beta Testing**: 5-10 external players identify flow and clarity issues
3. **Gamma Testing**: Measure timing, difficulty calibration, frustration points
4. **Launch Testing**: Full run-through with no more notes allowed

### Metrics to Track
- **Completion Rate**: Target 30-60% for 60-min rooms
- **Average Solve Time**: Per puzzle and total room
- **Hint Usage**: How often teams need hints, which puzzles trigger hints
- **Player Feedback**: Emotional response, engagement, confusion points

### Common Playtest Issues
- Solution ambiguity (multiple valid approaches cause confusion)
- Hidden hint syndrome (clue too subtle, players stuck without realizing)
- Bottleneck puzzles (one puzzle blocks entire team)
- Theme-mechanic disconnect (puzzle doesn't fit narrative)

## 7.4 Safety & Accessibility Standards

### Physical Safety
- Clear pathways free of trip hazards
- No exposed wiring or electrical hazards
- Proper ventilation for any fog/machines
- Emergency lighting and exit visibility

### Content Warnings
- Claustrophobia warnings for tight spaces
- Darkness warnings for blackout sections
- Physical activity warnings for moving elements
- Content advisories for horror/intense themes

### Accessibility Considerations
- Wheelchair-accessible pathways
- Alternative solutions for mobility-limited players
- Color-blind friendly puzzle elements
- Audio descriptions for hearing-impaired players
- Reduced-sensory versions for neurodivergent players
