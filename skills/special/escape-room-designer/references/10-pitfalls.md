# Common Pitfalls & Anti-Patterns

## 10.1 High-Severity Anti-Patterns

### 🔴 Anti-Pattern 1: Puzzle-Only Design (No Story)
```
❌ BAD: Creating room with "cool puzzles" connected by no narrative
✅ GOOD: Every puzzle reveals story; story explains why puzzle exists
```
**Why it matters**: Players remember the experience, not just puzzles. Story creates emotional investment.

### 🔴 Anti-Pattern 2: Ambiguous Solutions
```
❌ BAD: Puzzle can be solved multiple ways, but only one way triggers the next step
✅ GOOD: Test extensively; any valid solution should work
```
**Why it matters**: Players feel cheated when "correct" solution is arbitrary. They'll stop trusting your puzzles.

### 🔴 Anti-Pattern 3: Single Point of Failure
```
❌ BAD: One puzzle blocks entire room progress
✅ GOOD: Provide multiple paths, parallel puzzles, or backup hint system
```
**Why it matters**: When players stuck here, entire experience derails. One bottleneck kills flow.

### 🔴 Anti-Pattern 4: Ignoring Target Audience
```
❌ BAD: Designing for "experienced escape room players" when most clients are beginners
✅ GOOD: Know your demographic; calibrate difficulty and theme appropriately
```
**Why it matters**: Wrong difficulty = frustration for beginners or boredom for experts.

## 10.2 Medium-Severity Anti-Patterns

### 🟡 Anti-Pattern 5: Hint System as Band-Aid
```
❌ BAD: Hints added late to cover unclear design
✅ GOOD: Hints enhance experience, not fix broken puzzles; design with clarity in mind
```

### 🟡 Anti-Pattern 6: Theme-Mechanic Mismatch
```
❌ BAD: Solving a puzzle in a "1920s speakeasy" using a digital app
✅ GOOD: Keep solutions thematically appropriate; props should feel period-accurate
```

### 🟡 Anti-Pattern 7: Neglecting Reset Time
```
❌ BAD: Designing puzzles that take 10+ minutes to reset between groups
✅ GOOD: Plan for realistic turnover times; design reset into puzzle mechanism
```

### 🟡 Anti-Pattern 8: No Playtesting
```
❌ BAD: Launching room without external testing
✅ GOOD: Minimum 3 rounds of external playtesting before launch
```

## 10.3 Best Practices Checklist

### Before Design
- [ ] Define target audience: Who is this room for?
- [ ] Establish theme and emotional arc
- [ ] Map narrative beats to room sections

### During Design
- [ ] Every puzzle serves story
- [ ] Multiple solutions where possible
- [ ] Parallel puzzle paths to avoid bottlenecks
- [ ] Hint system integrated, not added later
- [ ] Flow tested with players from target audience

### Before Launch
- [ ] Playtested with 10+ external players
- [ ] Completion rate acceptable (30-60% typical target)
- [ ] Reset time realistic for business model
- [ ] Safety inspection passed
- [ ] Staff trained on troubleshooting

## 10.4 Red Flags to Watch For

| Red Flag | Risk Level | Action |
|----------|------------|--------|
| 80%+ failure rate on single puzzle | High | Redesign with more accessible clues |
| Players consistently confused about what to do | High | Add more contextual hints or clearer design |
| Theme changes mid-room (inconsistent) | Medium | Unify design language throughout |
| Puzzle solutions don't fit theme | Medium | Redesign to maintain immersion |
| No backup if puzzle fails | High | Add hint system or parallel solution path |
