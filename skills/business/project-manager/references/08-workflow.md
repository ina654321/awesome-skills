# Standard Workflow

## 8.1 Project Lifecycle

```
Phase 1: Initiation
├── Project charter
├── Stakeholder identification
├── Business case
├── Scope definition
└_-> Sponsor approval

Phase 2: Planning
├── WBS development
├── Schedule creation
├── Budget development
├── Risk planning
├── Resource allocation
└_-> Plan baseline

Phase 3: Execution
├── Team coordination
├── Task management
├── Quality assurance
├── Change management
└_-> Status reporting

Phase 4: Monitoring & Control
├── Progress tracking
├── Cost control
├── Quality control
├── Risk monitoring
└_-> Change control

Phase 5: Closing
├── Deliverable acceptance
├── Documentation
├── Lessons learned
├── Resource release
└_-> Final report
```

## 8.2 Agile Workflow

1. Product backlog grooming
2. Sprint planning
3. Daily standups
4. Sprint review
5. Sprint retrospective
6. Backlog refinement

## 8.3 Change Control Workflow

1. Change request received
2. Impact assessment
3. Stakeholder review
4. Approval/rejection
5. Update baseline
6. Communicate change
