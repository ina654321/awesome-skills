# UX Designer — Glossary

## Design Thinking Terms

**Empathy Map** — Visual tool for capturing user thoughts, feelings, sayings, and doings.

**Persona** — Fictional character representing a user segment. Based on research.

**Journey Map** — Visual representation of user experience over time, across touchpoints.

**Service Blueprint** — Diagram showing service delivery components and interactions.

**Jobs-to-be-Done (JTBD)** — Framework focusing on what users want to accomplish.

**How Might We (HMW)** — Question format for generating design opportunities.

## UX Research Terms

**Qualitative Research** — Open-ended, exploratory methods. Interviews, observations, focus groups.

**Quantitative Research** — Structured methods with measurable data. Surveys, analytics, A/B tests.

**Generative Research** — Research to discover new opportunities and insights.

**Evaluative Research** — Research to test and validate existing solutions.

**Moderated Testing** — Researcher guides participant through tasks.

**Unmoderated Testing** — Participant completes tasks without researcher present.

**Think-Aloud Protocol** — Participant verbalizes thoughts while performing tasks.

**Probing** — Follow-up questions to explore participant responses.

**Affinity Diagramming** — Clustering insights into themes.

**Cognitive Walkthrough** — Expert review of usability based on user mental models.

## UI Design Terms

**Wireframe** — Low-fidelity layout showing structure and content placement.

**Mockup** — High-fidelity visual design without interactivity.

**Prototype** — Interactive model of design for testing.

**Lo-fi** — Low-fidelity: sketches, paper prototypes.

**Hi-fi** — High-fidelity: visual design, close to final product.

**Design Token** — Named value representing design decision (color, spacing, typography).

**Component** — Reusable UI element with defined behavior and states.

**State** — Condition of a component (default, hover, active, disabled, error, loading).

## Visual Design Terms

**Visual Hierarchy** — Order of importance of elements based on visual cues.

**Gestalt Principles** — Principles of visual perception: proximity, similarity, closure, etc.

**Whitespace** — Empty space around and between elements.

**Alignment** — Arrangement of elements along a common line or axis.

**Contrast** — Visual difference between elements for emphasis or readability.

**Balance** — Distribution of visual weight in a design.

**Rhythm** — Repeating pattern that creates visual flow.

## Accessibility Terms

**WCAG** — Web Content Accessibility Guidelines. Standards for web accessibility.

**Screen Reader** — Software that reads screen content aloud. NVDA, JAWS, VoiceOver.

**ARIA** — Accessible Rich Internet Applications. Attributes for accessible dynamic content.

**Alt Text** — Text description of image for screen readers.

**Focus Order** — Sequence in which interactive elements receive keyboard focus.

**Skip Link** — Hidden link allowing keyboard users to skip navigation.

**Landmark Roles** — ARIA roles identifying page regions (banner, main, navigation).

**Color Blindness** — 8% of men, 0.5% of women. Types: protanopia, deuteranopia.

## Layout & Navigation Terms

**Information Architecture (IA)** — Structure and organization of content.

**Sitemap** — Hierarchical list of all pages in a product.

**Navigation** — System for moving between pages and sections.

**Breadcrumb** — Secondary navigation showing location in hierarchy.

**Pagination** — Dividing content into pages.

**Card Layout** — Content organized into discrete rectangular containers.

**Grid System** — Framework of intersecting horizontal and vertical lines.

**Breakpoint** — Screen width at which layout adapts.

**Responsive Design** — Design that adapts to different screen sizes.

**Mobile-First** — Designing for mobile before desktop.

## Interaction Design Terms

**Interaction Pattern** — Common solution to common UI problem.

**Microinteraction** — Small, focused moment of interaction (toggle, button feedback).

**State Change** — Visual change when component condition changes.

**Transition** — Animation between states or views.

**Loading State** — Visual feedback during asynchronous operations.

**Error State** — Visual feedback when something goes wrong.

**Success State** — Visual feedback confirming action completion.

**Empty State** — Content shown when no data exists yet.

## User Behavior Terms

**Fitts's Law** — Time to reach target = distance/size. Big, close targets are easier.

**Hick's Law** — Decision time increases with number of choices.

**Jakob's Law** — Users prefer your site works like all other sites.

**Aesthetic-Usability Effect** — aesthetically pleasing design perceived as more usable.

**Von Restorff Effect** — Item that stands out is more memorable.

**Serial Position Effect** — First and last items in a list are remembered best.

**Cognitive Load** — Amount of mental effort required to process information.
