# Scenario Examples

## 9.1 Data Breach Response

**User:** "We've detected unauthorized access to our database. What's the response plan?"

**Expert:**
> **Breach Response:**
> 
> 1. **Containment**:
>    - Isolate affected systems
>    - Preserve evidence
>    - Reset credentials
> 
> 2. **Assessment**:
>    - Determine scope
>    - Identify data exposed
>    - Assess impact
> 
> 3. **Notification**:
>    - Report to regulators (72 hrs)
>    - Notify affected customers
>    - File required reports

## 9.2 Payment Integration Security

**User:** "Building payment integration with third-party processor. What security controls needed?"

**Expert:**
> **Payment Security:**
> 
> 1. **Tokenization**:
>    - Use payment tokens
>    - Never store card data
>    - Implement PCI DSS
> 
> 2. **API Security**:
>    - OAuth/API keys
>    - Encryption in transit
>    - Rate limiting
>    - Input validation
> 
> 3. **Monitoring**:
>    - Transaction monitoring
>    - Fraud detection
>    - Logging and alerting

## 9.3 Algorithm Bias

**User:** "Our lending algorithm shows potential discrimination. How to handle?"

**Expert:**
> **Algorithm Fairness:**
> 
> 1. **Assessment**:
>    - Run fairness tests
>    - Identify protected classes
>    - Measure impact
> 
> 2. **Remediation**:
>    - Review feature selection
>    - Adjust model
>    - Add fairness constraints
> 
> 3. **Governance**:
>    - Document approach
>    - Regular audits
    - Board reporting
