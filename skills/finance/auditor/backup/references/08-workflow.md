# Standard Workflow

## 8.1 Audit Process

```
Phase 1: Planning
├── Client acceptance
├── Team composition
├── Preliminary engagement planning
├── Understanding the entity
├── Risk assessment
└── Materiality determination

Phase 2: Internal Controls
├── Control identification
├── Control testing design
├── IT general controls
├── Application controls
└── SOX 404 testing

Phase 3: Substantive Testing
├── Account balance testing
├── Transaction testing
├── Analytical procedures
├── Substantive procedures
└── Journal entry testing

Phase 4: Completion
├── Subsequent events review
├── Going concern evaluation
├── Written representations
├── Completion memorandum
└── Audit committee communication

Phase 5: Reporting
├── Auditor report preparation
├── Opinion formulation
├── Communication with those charged
└── Delivery and filing
```

## 8.2 Financial Statement Audit Workflow

1. Plan the audit and document understanding
2. Identify and assess risks
3. Determine response to risks
4. Perform procedures
5. Evaluate audit evidence
6. Communicate findings
7. Issue audit opinion

## 8.3 Internal Control Audit Workflow

1. Understand control environment
2. Assess control risk
3. Test design effectiveness
4. Test operating effectiveness
5. Evaluate deficiencies
6. Report material weaknesses

## 8.4 SOX 404 Workflow

1. Top-down risk assessment
2. Identify controls
3. Document controls
4. Test controls
5. Remediate deficiencies
6. Report on internal control
