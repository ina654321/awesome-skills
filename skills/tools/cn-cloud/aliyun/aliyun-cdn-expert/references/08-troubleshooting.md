# Troubleshooting

## 8.1 访问异常

### 8.1.1 源站无法访问 (502/504)

**症状:** CDN节点返回502 Bad Gateway或504 Gateway Timeout

**排查步骤:**
1. 登录CDN控制台 → 域名管理 → 点击域名
2. 查看"回源统计"判断源站响应时间
3. 测试源站IP直接访问是否正常
4. 检查源站安全组/防火墙是否阻断了CDN回源IP段

**解决方案:**
```bash
# 测试源站连通性
curl -I http://源站IP/

# 检查CDN回源IP段(需要放行)
# 阿里云CDN回源IP段: 39.96.0.0/12, 47.96.0.0/12 等
```

### 8.1.2 缓存未生效 (200/304异常)

**症状:** 同一资源每次访问都回源，未命中缓存

**排查步骤:**
1. 检查Cache-Control是否设置为no-cache/no-store/private
2. 验证源站是否返回Vary: *头
3. 确认缓存规则是否覆盖该文件类型
4. 检查浏览器F12 Network中X-Cache头

**解决方案:**
- 静态资源设置: `Cache-Control: max-age=31536000`
- 在CDN控制台配置缓存过期时间规则
- 开启"过滤参数"功能

### 8.1.3 资源 404 Not Found

**症状:** CDN返回404，但源站文件存在

**排查步骤:**
1. 确认CDN域名CNAME已正确解析
2. 检查源站路径与CDN回源路径是否一致
3. 验证文件是否在黑名单/ Referer访问控制中被拦截

## 8.2 HTTPS问题

### 8.2.1 SSL证书配置失败

**症状:** HTTPS访问时证书无效或过期

**排查步骤:**
1. 确认证书与域名匹配
2. 检查证书链是否完整
3. 验证证书格式(PEM格式)

**解决方案:**
```bash
# 检查证书有效期
openssl x509 -in cert.pem -noout -dates

# 验证证书链
openssl verify -CAfile ca-bundle.crt cert.pem
```

### 8.2.2 混合内容警告

**症状:** HTTPS页面加载HTTP资源被浏览器阻止

**解决方案:**
- 源站配置强制HTTPS回源
- 将页面中所有HTTP链接改为HTTPS
- 开启CDN的"协议跟随回源"功能

## 8.3 性能问题

### 8.3.1 命中率低

**常见原因:**
- 大量不可缓存的动态请求
- 缓存规则过于严格
- 缓存key设计不合理

**解决方案:**
- 开启过滤参数或配置忽略参数
- 延长静态资源缓存时间
- 使用"只看速度"模式跳过缓存检测

### 8.3.2 回源带宽高

**排查:**
1. 查看TOP URL找出高回源请求
2. 检查是否存在大量不可缓存的资源
3. 评估源站服务器承载能力

**解决方案:**
- 增加缓存规则覆盖更多资源
- 使用预热功能提前缓存热门资源
- 考虑升级源站带宽
