# Troubleshooting

## 8.1 函数调用问题

### 8.1.1 函数调用超时

**症状:** 函数执行报504 Gateway Timeout

**排查步骤:**
1. 检查函数超时设置
2. 查看函数执行日志
3. 分析函数执行耗时

```bash
# 查看日志
# 函数计算控制台 → 函数管理 → 日志查询
# 或使用CLI
fc log project=/ acs/logstore/triggers
```

**解决方案:**
- 增加超时时间(最大600秒)
- 检查代码是否有死循环或长时间等待
- 优化冷启动时间(使用轻量依赖)
- 开启预置并发

### 8.1.2 冷启动延迟高

**原因:** 函数实例需要初始化，依赖越多越慢

**解决方案:**
- 精简代码包体积
- 使用预置并发提前启动实例
- 减少依赖层(Layer)
- 选择合适的运行时版本

## 8.2 触发器问题

### 8.2.1 OSS触发器不触发

**排查步骤:**
1. 确认触发器已正确配置
2. 检查OSS Bucket事件类型是否匹配
3. 验证函数权限(RAM角色)

```json
{
  "events": ["oss:ObjectCreated:PutObject", "oss:ObjectCreated:PostObject"],
  "filter": {
    "key": {
      "prefix": "images/",
      "suffix": [".jpg", ".png"]
    }
  }
}
```

**解决方案:**
- 确保函数有OSS触发器的触发权限
- 检查触发器配置中的prefix/suffix过滤条件
- 确认Bucket和函数在同一区域

### 8.2.2 定时触发器不执行

**原因:** Cron表达式格式错误或时区问题

**解决方案:**
```bash
# Cron表达式格式: 秒 分 时 日 月 周
# 每5分钟: 0 */5 * * * *
# 每天凌晨2点: 0 0 2 * * *

# 注意: 阿里云使用UTC时区
# 北京时间需要-8小时
# 北京时间每天2点 = UTC前一天18点
# 所以Cron应为: 0 0 18 * * *
```

## 8.3 权限问题

### 8.3.1 函数无权限访问其他服务

**症状:** AccessDenied错误

**解决方案:**
1. 为函数配置RAM角色
2. 添加相应的权限策略

```json
{
  "Version": "1",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": ["oss:GetObject", "oss:PutObject"],
      "Resource": "acs:oss:*:*:my-bucket/*"
    }
  ]
}
```

## 8.4 计费异常

### 8.4.1 费用超出预期

**常见原因:**
- 函数调用次数异常(被人恶意调用)
- 执行时长超出预期
- 公网流量未限制

**解决方案:**
- 设置函数级别的并发限制
- 配置日志消费上限
- 开启API网关的认证和限流
- 设置函数内存上限
