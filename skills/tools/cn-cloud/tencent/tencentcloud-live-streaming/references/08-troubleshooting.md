# Troubleshooting

## 8.1 推流问题

### 8.1.1 OBS推流失败

**症状:** OBS显示"连接失败"或"正在重连"

**排查步骤:**
1. 确认推流地址和密钥正确
2. 检查网络是否可访问推流域名
3. 确认推流域名未被禁用

```bash
# 测试推流域名连通性
telnet push.yourdomain.com 1935

# 检查推流端口是否被防火墙阻断
# 腾讯云控制台 → 直播 → 域名管理 → 鉴权配置
```

**解决方案:**
- 在直播控制台重新生成推流地址
- 检查OBS的编码设置(码率不超过限制)
- 确认推流域名已启用

### 8.1.2 推流卡顿

**原因:** 码率过高或网络上行带宽不足

**解决方案:**
- 降低视频码率(建议2500-4000kbps)
- 降低帧率(建议15-20fps)
- 优先使用有线网络

## 8.2 播放问题

### 8.2.1 播放404

**排查:**
1. 确认流是否正在推送
2. 检查播放域名配置
3. 验证防盗链签名

```bash
# 测试播放地址
ffplay "rtmp://pull.yourdomain.com/live/stream?token=xxx"
```

### 8.2.2 延迟过高

**原因:** 链路缓存或编码参数问题

**解决方案:**
- 播放协议: RTMP延迟 < FLV < HLS
- 使用低延迟HLS: 调整切片大小
- 开启快直播(基于WebRTC)

## 8.3 录制问题

### 8.3.1 录制文件不存在

**排查:**
1. 确认录制功能已开启
2. 检查录制模板配置
3. 查看COS存储权限

**解决方案:**
- 检查录制回调通知
- 确认COS Bucket存在且有写入权限
