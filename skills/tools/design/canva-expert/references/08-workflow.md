# Standard Workflow

## 8.1 Getting Started

### Basic Design Creation Workflow

```
Phase 1: Concept
├── Define purpose (social post, presentation, print)
├── Check brand guidelines
├── Choose dimensions or start from template
└── Gather brand assets from Brand Kit

Phase 2: Build
├── Add background (color, gradient, or image)
├── Place logo (pre-sized, locked position)
├── Arrange text elements (headline, subhead, body)
├── Add supporting graphics or photos
└── Apply brand colors throughout

Phase 3: Refine
├── Adjust spacing and alignment
├── Check hierarchy and readability
├── Verify brand consistency
├── Preview on different device sizes
└── Request feedback if collaborative

Phase 4: Deliver
├── Export in required format(s)
├── Name file with clear convention
├── Share or publish directly
└── Archive or duplicate for variations
```

### Brand-Consistent Design Workflow

```
Phase 1: Brand Kit Setup
├── Upload brand colors (hex codes)
├── Add brand fonts (upload or select)
├── Store logo files (PNG, SVG)
└── Create brand template designs

Phase 2: Template Creation
├── Create base design for each type
├── Lock brand elements (logo, colors, fonts)
├── Leave editable zones for content
└── Save as team template

Phase 3: Content Creation
├── Start from brand template
├── Replace placeholder content
├── Swap images with brand assets
└── Adjust only editable elements

Phase 4: Approval
├── Share for team review
├── Collect feedback in comments
├── Make revisions
└── Approve and export
```

## 8.2 Asset Production Workflow

### Social Media Content Pipeline

1. **Batch Planning**
   - Create content calendar in spreadsheet
   - List all posts with dates, platforms, sizes
   - Gather all content assets upfront

2. **Template Setup**
   - Create one design per size format
   - Set up Bulk Create data source
   - Map fields for dynamic content

3. **Bulk Generation**
   - Import data (Google Sheets or CSV)
   - Preview generated designs
   - Generate all variations
   - Review for errors

4. **Quality Check**
   - Review sample of generated designs
   - Verify text fits properly
   - Check image placements
   - Correct any issues

5. **Export & Schedule**
   - Download all designs
   - Upload to scheduling tool
   - Schedule posts with platform tags

### Presentation Workflow

```
Step 1: Setup
├── Create presentation with master slide
├── Apply brand colors and fonts
├── Design title slide and section dividers
└── Create content slide templates

Step 2: Content Fill
├── Add text content to each slide
├── Insert charts and graphs
├── Add speaker notes
└── Place images and icons

Step 3: Polish
├── Apply consistent animations
├── Add transitions between slides
├── Review all slides in presentation mode
└── Proofread all text

Step 4: Export
├── Download as PDF or PPTX
├── Set slides as presentation or handout view
└── Share with stakeholders
```

### Print Design Workflow

1. **Document Setup**
   - Choose print-ready template or create custom
   - Set exact dimensions including bleed (3mm)
   - Choose CMYK color mode (via export)

2. **Design Phase**
   - Place images at full resolution
   - Use CMYK color values for accuracy
   - Keep text away from trim edges
   - Add crop marks if exporting manually

3. **Pre-flight Check**
   - Verify all fonts are loaded
   - Confirm images are high resolution (300 DPI+)
   - Check bleed is included
   - Review CMYK values

4. **Export for Print**
   - Choose PDF Print format
   - Enable bleed and crop marks
   - Select CMYK color conversion
   - Download and send to printer

## 8.3 Collaboration Workflow

### Team Design Workflow

```
Phase 1: Organization
├── Create team folders by project/category
├── Set folder permissions (view/edit)
├── Establish naming conventions
└── Document folder structure

Phase 2: Roles
├── Admin: Full access, manage team, billing
├── Editor: Create, edit, delete designs
├── Viewer: View and comment only
└── Specific: Folder-level permissions

Phase 3: Creation
├── Assign design ownership
├── Create in shared folder
├── Use shared Brand Kit
└── Apply team templates

Phase 4: Review
├── Share via link or email
├── Enable commenting
├── Collect feedback in comments
└── Track version history

Phase 5: Approval
├── Request approval from stakeholders
├── Make final revisions
├── Lock design when approved
└── Archive original versions
```

### Client Collaboration

1. **Share Design**
   - Click Share button
   - Set "Anyone with link" or specific emails
   - Choose "Can view" or "Can edit"
   - Enable commenting

2. **Collect Feedback**
   - Client adds comments directly on design
   - Reply to comments to clarify
   - Mark comments as resolved
   - Track changes in version history

3. **Make Revisions**
   - Duplicate design as new version
   - Make changes based on feedback
   - Share updated link
   - Repeat until approved

4. **Final Delivery**
   - Download final approved version
   - Export in requested format
   - Transfer to client folder or platform
   - Document approved version

### Version Management

| Practice | Implementation |
|----------|----------------|
| Auto-versioning | Canva saves automatically |
| Manual versions | Duplicate design with new name |
| Naming | Use dates or version numbers |
| Archiving | Move old designs to Archive folder |
| Recovery | Access from Trash within 30 days |
