# Standard Workflow

## 8.1 Coding Workflow

```
Phase 1: Record Review
├── Identify encounter type
├── Verify provider documentation
├── Read H&P and progress notes
├── Review orders and procedures
└── Note all diagnoses documented

Phase 2: Code Assignment
├── Assign principal diagnosis
├── Assign secondary diagnoses
├── Assign procedure codes
├── Apply coding guidelines
├── Sequence correctly
└── Query provider if unclear

Phase 3: Quality Review
├── Verify code accuracy
├── Check bundling rules
├── Ensure modifier use
├── Validate DRG assignment
└── Complete within timeframe
```

## 8.2 Inpatient Coding

```
Coding Steps:
├── Review face sheet
├── Read history and physical
├── Follow progress notes
├── Review discharge summary
├── Check operative reports
└── Verify final diagnosis

DRG Logic:
├── Principal diagnosis
├── Secondary diagnoses
├── Procedures performed
├── Presence of MCC/CC
└── Complication codes
```

## 8.3 Outpatient Coding

```
Encounter Types:
├── Office visits (99202-99215)
├── Hospital outpatient (99281-99285)
├── Emergency department (99281-99285)
├── Observation (99217-99220)
└── Outpatient surgery (CPT-based)

Key Elements:
├── Level of service supported
├── Procedure codes linked
├── Modifiers applied
├── Preventive vs problem
└── Medical decision making
```

## 8.4 Query Workflow

```
Query Triggers:
├── Unclear or conflicting diagnoses
├── Missing complication information
├── Possible query needed for specificity
├── Atypical treatment patterns
└── Significant clinical indicators

Query Format:
├── Clinically indicated
├── Non-leading questions
├── Multiple choice options
├── Physician response documented
└── Retained in record
```
