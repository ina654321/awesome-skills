# Standard Workflow

## 8.1 Product Requirements Gathering

```
Phase 1: User Research
├── Conduct focus groups with elderly users
├── Interview caregivers and family members
├── Shadow daily activities in care settings
├── Document pain points and needs
└── Analyze competitive products

Phase 2: Regulatory Planning
├── Determine device classification (FDA)
├── Identify applicable standards (IEC 62366, ISO 14971)
├── Plan usability engineering file
└── Design for accessibility requirements

Phase 3: Design Requirements
├── Define user needs and intended use
├── Translate to design inputs
├── Establish user interface requirements
├── Specify environmental conditions
└── Document use-related risks
```

## 8.2 Accessibility Design

```
Core Accessibility Principles (WCAG 2.1):
├── Perceivable: Large text, high contrast, audio descriptions
├── Operable: Simple controls, voice activation, large targets
├── Understandable: Clear language, intuitive navigation
└── Robust: Works with assistive technologies

Elder-Specific Design:
├── Reduced fine motor requirements
├── Compensation for vision/hearing loss
├── Cognitive load minimization
├── Error prevention and recovery
└── Physical ergonomics
```

## 8.3 Clinical Validation Workflow

```
Step 1: Formative Usability Testing
├── Prototype testing with target users
├── Identify critical use errors
├── Iterate design based on feedback
└── Document all findings

Step 2: Summative Testing
├── Validation with representative users
├── Document all use errors and hazards
├── Verify mitigations are effective
└── Complete usability engineering file

Step 3: Post-Market Surveillance
├── Monitor adverse events
├── Collect user feedback
├── Implement design improvements
└── Maintain risk management file
```

## 8.4 Regulatory Pathways

| Pathway | Device Class | Requirements |
|---------|--------------|---------------|
| 510(k) | Class II | Predicate comparison, performance testing |
| PMA | Class III | Clinical trials, safety/efficacy data |
| De Novo | Class I/II | No predicate, risk-based approach |
| 510(k) Exempt | Class I | General controls only |
