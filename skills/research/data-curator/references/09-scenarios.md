# Research Terminology Glossary

## A

- **API** — Application Programming Interface; standardized data access method
- **Archive** — Long-term storage with preservation commitment
- **Audit trail** — Documented history of data creation, modification, access

## C

- **Checksums** — Digital fingerprints (MD5, SHA-256) for file integrity
- **Controlled vocabulary** — Standardized terms ensuring consistent metadata
- **CRediT taxonomy** — 14 roles defining scholarly contributions

## D

- **DataCite** — Organization assigning DOIs for datasets
- **Data curation** — Active management throughout data lifecycle
- **Data dictionary** — Documentation of variables, codes, formats
- **Data enclave** — Secure environment for analyzing sensitive data
- **Data lineage** — Provenance and transformation documentation
- **Data management plan (DMP)** — Formal document outlining data procedures
- **Data use agreement** — Legal contract governing data use
- **De-identification** — Removing personally identifiable information
- **DOI** — Digital Object Identifier; persistent scholarly reference

## F

- **FAIR** — Findable, Accessible, Interoperable, Reusable principles
- **FAIRification** — Making data compliant with FAIR principles

## I

- **Identifier** — Unique string referencing a specific data object
- **Immutable storage** — Storage preventing modification
- **Ingest** — Transferring data into repository

## K

- **k-anonymity** — Privacy protection ensuring each record matches k-1 others

## M

- **Metadata** — Structured information describing data
- **MOU** — Memorandum of Understanding for data sharing

## O

- **ORCID** — Persistent identifier for researchers
- **OAI-PMH** — Protocol for metadata harvesting

## P

- **PID** — Persistent Identifier for digital objects
- **Pre-registration** — Publishing study design before data collection
- **Provenance** — Data origin and processing history

## R

- **RDA** — Research Data Alliance; develops data sharing standards
- **Registry** — Searchable catalog of research resources
- **ReID (Re-identification)** — Linking anonymous data to individuals
- **Repository** — Infrastructure for storage and access

## S

- **Schema** — Formal definition of data structure
- **Sensitive data** — Information requiring special handling
- **SOP** — Standard Operating Procedure
