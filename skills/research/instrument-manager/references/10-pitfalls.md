# Instrument Manager Templates, Checklists & Protocols

Practical tools for managing scientific instrumentation throughout the lifecycle.

## Instrument Acquisition Checklist

Before purchasing new equipment:

- [ ] Conduct user needs assessment; survey current and anticipated research needs
- [ ] Review existing equipment inventory to avoid duplication
- [ ] Evaluate instrument options from multiple vendors
- [ ] Request and evaluate vendor demonstrations using standard test protocols
- [ ] Obtain quotes from at least 3 vendors (if available)
- [ ] Assess total cost of ownership (purchase, installation, service contract, consumables, disposal)
- [ ] Verify compatibility with existing infrastructure (power, network, space, ventilation)
- [ ] Verify space dimensions and weight meet facility requirements
- [ ] Confirm electrical requirements (voltage, amperage, single/three-phase)
- [ ] Assess network connectivity and data transfer requirements
- [ ] Coordinate with facilities management for site preparation
- [ ] Verify environmental conditions (temperature, humidity, vibration) are suitable
- [ ] Check for regulatory requirements (radiation license, biosafety containment)
- [ ] Negotiate warranty terms and service contract pricing before issuing PO
- [ ] Verify manufacturer support availability in your region
- [ ] Confirm spare parts availability for the expected instrument lifetime
- [ ] Obtain intellectual property and data ownership terms
- [ ] Verify software licensing requirements and costs
- [ ] Document decision rationale and approval process
- [ ] Register equipment in institutional asset management system upon receipt

## Installation & Qualification Protocol

When installing new equipment:

**Pre-Installation:**
- [ ] Verify site preparation is complete (power, network, gas, ventilation)
- [ ] Confirm environmental conditions meet manufacturer specifications
- [ ] Verify receiving area is clear and accessible for delivery
- [ ] Coordinate with receiving/institutional shipping for delivery
- [ ] Obtain manufacturer pre-installation documentation

**Installation:**
- [ ] Unpack instrument per manufacturer instructions
- [ ] Verify all components match the order manifest
- [ ] Document serial numbers, model numbers, and software versions
- [ ] Photograph instrument and packaging upon arrival
- [ ] Complete Installation Qualification (IQ) per manufacturer protocol:
  - [ ] Verify instrument is correctly installed
  - [ ] Verify utilities (power, gas, water) are connected correctly
  - [ ] Verify environmental conditions are within spec
  - [ ] Document all installation steps

**Operational Qualification (OQ):**
- [ ] Perform manufacturer-specified startup procedures
- [ ] Verify system powers on and initializes correctly
- [ ] Test alarm and safety systems per protocol
- [ ] Verify software installation and licensing
- [ ] Test basic operation using manufacturer reference materials
- [ ] Verify data acquisition and transfer to LIMS/ELN/network
- [ ] Document all OQ test results and acceptance criteria

**Performance Qualification (PQ):**
- [ ] Test instrument performance using real samples or reference materials
- [ ] Verify performance meets manufacturer specifications
- [ ] Establish routine QC parameters and control limits
- [ ] Train designated users and verify competency
- [ ] Approve instrument for production use
- [ ] Archive installation and qualification records

## Preventive Maintenance Schedule Template

### Monthly PM Tasks:
- [ ] Inspect instrument for physical damage, loose connections, wear
- [ ] Clean exterior surfaces and accessible components per manufacturer SOP
- [ ] Check consumable levels (gases, solvents, reagents)
- [ ] Review QC data for trends; assess performance
- [ ] Verify data backup and archival systems are functioning
- [ ] Check and clean filters, traps, and septa as applicable
- [ ] Inspect cables, tubing, and connectors for wear

### Quarterly PM Tasks (in addition to monthly):
- [ ] Perform manufacturer-specified quarterly maintenance
- [ ] Verify calibration against reference standards
- [ ] Update software patches and firmware if validated
- [ ] Review and update maintenance logs
- [ ] Assess spare parts inventory
- [ ] Evaluate upcoming warranty/service contract renewal
- [ ] Review user satisfaction and instrument utilization data

### Annual PM Tasks (in addition to quarterly):
- [ ] Comprehensive performance qualification
- [ ] Full calibration traceable to NIST standards
- [ ] Service contract review and renewal negotiation
- [ ] Evaluate technology obsolescence and replacement planning
- [ ] Conduct comprehensive safety inspection
- [ ] Review and update all SOPs
- [ ] Update training records and competency assessments
- [ ] Submit equipment for regulatory inspections if applicable

## Quality Control Protocol

For analytical instruments (spectrophotometers, chromatographs, mass spectrometers):

**Daily QC:**
- [ ] Verify instrument has completed startup and warm-up procedures
- [ ] Run system suitability / instrument suitability checks per method
- [ ] Analyze at least two levels of QC material (low, high)
- [ ] Verify QC results are within established control limits
- [ ] Document all QC results in LIMS or QC log
- [ ] Do not proceed with sample analysis if QC fails

**QC Failure Response:**
- [ ] Repeat QC to rule out transcription or preparation error
- [ ] Check reagent and standard expiration dates
- [ ] Verify calibrator preparation and integrity
- [ ] Perform instrument diagnostic procedures per manufacturer
- [ ] Perform corrective maintenance if qualified to do so
- [ ] If unresolved, escalate to service engineer
- [ ] Document root cause and corrective actions
- [ ] Re-verify QC before resuming sample analysis
- [ ] Assess whether affected samples need reanalysis

**Weekly / Periodic QC:**
- [ ] Review QC charts for trends (drift, shifts, increasing variability)
- [ ] Evaluate proficiency testing or inter-laboratory comparison results
- [ ] Assess whether control limits need adjustment based on data
- [ ] Review audit trail for any anomalies
- [ ] Verify data backup and archive processes are running

## Training & Competency Assessment Protocol

Before granting independent instrument access:

**Training Requirements:**
- [ ] Complete general laboratory safety training
- [ ] Complete instrument-specific safety training (chemical, cryogenic, radiation hazards)
- [ ] Review instrument SOP and manufacturer documentation
- [ ] Observe qualified user performing procedures
- [ ] Perform hands-on operation under direct supervision
- [ ] Pass competency assessment demonstrating:
  - [ ] Proper startup and shutdown procedures
  - [ ] Routine operation per SOP
  - [ ] QC procedures and documentation
  - [ ] Data acquisition, analysis, and storage
  - [ ] Basic troubleshooting and problem recognition
  - [ ] Emergency procedures
  - [ ] Waste disposal procedures
  - [ ] Proper PPE use

**Competency Assessment:**
- [ ] Document training completion date, trainer name, and methods used
- [ ] Document assessment results (satisfactory/unsatisfactory)
- [ ] Retrain and reassess if competency not demonstrated
- [ ] Schedule periodic competency reassessment (typically annual)
- [ ] Update access permissions based on competency verification
- [ ] Maintain training records per institutional and regulatory requirements

## Instrument Decommissioning & Disposal Protocol

When removing equipment:

1. **Data Management:**
   - [ ] Back up and archive all data per institutional policy
   - [ ] Transfer data ownership as needed
   - [ ] Close out data in LIMS; update instrument status
   - [ ] Verify data retention meets regulatory and sponsor requirements

2. **Decontamination:**
   - [ ] Review instrument hazards (chemical, biological, radioactive, cryogenic)
   - [ ] Follow institutional decontamination procedures per hazard type
   - [ ] Obtain decontamination certification from EHS department
   - [ ] Document decontamination procedures performed
   - [ ] Remove biohazard/radiation warning labels after verification

3. **Regulatory Compliance:**
   - [ ] For radiation-producing equipment: notify radiation safety officer; complete surrender procedures
   - [ ] For lasers: notify laser safety officer; complete registration cancellation
   - [ ] For controlled substances equipment (DEA): complete required disposition
   - [ ] Update all regulatory registrations and licenses

4. **Asset Management:**
   - [ ] Remove from institutional asset inventory
   - [ ] Update insurance records
   - [ ] Process surplus or trade-in documentation
   - [ ] Update LIMS and facility management system records

5. **Disposal:**
   - [ ] Select certified electronic waste (e-waste) recycler for electronic equipment
   - [ ] Follow state environmental regulations for chemical/hazardous waste disposal
   - [ ] Document disposal method and certificate of destruction
   - [ ] Update records with disposal date and method

## Instrument Emergency Response Protocol

When an emergency occurs:

1. **Immediate Response:**
   - [ ] Ensure personal safety first; evacuate if necessary
   - [ ] Activate emergency shutdown procedures if trained and safe to do so
   - [ ] Alert building/floor emergency coordinator
   - [ ] Call emergency services (911) for injuries, fire, or significant spills
   - [ ] Do not re-enter until cleared by emergency personnel

2. **Instrument-Specific Emergencies:**
   - **Chemical Spill**: Evacuate; call EHS; use spill kit if trained and safe
   - **Cryogenic Burn**: Remove contaminated clothing; flush with water; seek medical attention
   - **Laser Exposure**: Seek immediate medical evaluation; report to supervisor and LSO
   - **Radiation Incident**: Notify RSO immediately; contain if possible; seek medical evaluation
   - **Electrical Shock**: Do not touch victim; disconnect power; call emergency services

3. **After the Emergency:**
   - [ ] Secure the instrument to prevent further use
   - [ ] Document incident in incident report system
   - [ ] Notify supervisor and institutional EHS
   - [ ] Investigate root cause
   - [ ] Implement corrective actions before resuming use
   - [ ] Update emergency procedures if gaps identified
