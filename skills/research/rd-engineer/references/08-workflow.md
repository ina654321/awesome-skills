# Standard Workflow

## 8.1 New Product Development Process

### Phase 1: Concept Phase (2-8 weeks)

**Step 1.1 - Voice of Customer (VOC)**
- Conduct interviews with 10-20 potential users
- Observe users in their actual environment
- Document pain points and unmet needs

**Step 1.2 - Requirements Translation**
- Convert customer language to technical requirements
- Prioritize requirements (MoSCoW: Must/Should/Could/Won't)
- Create product requirements document (PRD)

**Step 1.3 - Concept Generation**
- Generate minimum 3 concept alternatives
- Use brainstorming, TRIZ, or other ideation methods
- Create rough sketches or mockups

**Step 1.4 - Concept Screening**
- Apply Pugh matrix or decision matrix
- Evaluate against criteria: feasibility, cost, performance
- Select 1-2 concepts for feasibility study

### Phase 2: Feasibility Phase (4-12 weeks)

**Step 2.1 - Technical Feasibility**
- Build proof-of-concept prototypes
- Verify key technical assumptions
- Identify technical risks and mitigation

**Step 2.2 - Business Case**
- Estimate development cost and timeline
- Project market size and pricing
- Calculate ROI and break-even

**Step 2.3 - Risk Assessment**
- Conduct preliminary DFMEA
- Identify regulatory requirements
- Assess IP landscape

**Step 2.4 - Gate Review**
- Present findings to steering committee
- Go/No-Go decision on proceeding to development

### Phase 3: Development Phase (3-12 months)

**Step 3.1 - Detailed Design**
- Complete CAD models and drawings
- Create engineering specifications
- Finalize tolerance calls

**Step 3.2 - Prototype Build**
- Build functional prototypes (multiple iterations)
- Conduct design verification testing (DVT)
- Iterate based on test results

**Step 3.3 - DFMEA Completion**
- Complete failure mode analysis
- Implement design changes based on RPN
- Verify mitigations are effective

**Step 3.4 - Design for Manufacturability Review**
- DFM analysis with manufacturing team
- Design for assembly review
- Tooling design and procurement

### Phase 4: Validation Phase (2-6 months)

**Step 4.1 - Design Verification**
- Test to engineering specifications
- Confirm all requirements are met
- Document verification results

**Step 4.2 - Design Validation**
- Test with actual users (if applicable)
- Confirm product satisfies user needs
- Validate in use environment

**Step 4.3 - Regulatory**
- Complete required certifications
- Document for regulatory submission
- Obtain necessary approvals

**Step 4.4 - Production Validation**
- Process qualification (IQ/OQ/PQ)
- First article inspection
- Production pilot run

### Phase 5: Launch (Ongoing)

**Step 5.1 - Production Ramp-up**
- Scale production to volume
- Train production staff
- Establish quality controls

**Step 5.2 - Launch Support**
- Address field issues
- Gather customer feedback
- Document lessons learned

## 8.2 Problem Diagnosis Workflow

```
Step 1: Define the Problem
├── What failed? How did it fail? When?
└── Document symptoms precisely

Step 2: Gather Data
├── Failure analysis
├── Test results
└── Customer complaints

Step 3: Root Cause Analysis
├── 5 Whys
├── Fishbone diagram
└── Fault tree analysis

Step 4: Identify Countermeasures
├── Design change
├── Process change
└── Detection improvement

Step 5: Implement and Validate
├── Test the fix
├── Verify effectiveness
└── Update documentation

Step 6: Update Documentation
├── DFMEA
├── PFMEA
└── Control plans
```
