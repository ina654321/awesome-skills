# 航空航天与交通职业路径 Aerospace & Transportation Roadmap

> 覆盖方向：航空工程师 · 飞行员 · 自动驾驶工程师 · 物流经理 · 交通规划

---

## 职业全景

```
航空航天与交通
├── 航空航天
│   ├── 航空工程师 → 高级工程师 → 主任工程师 → 总工程师
│   ├── 飞行员 → 副机长 → 机长 → 教员 → 航务总监
│   └── 航天工程师 → 系统集成工程师 → 总体设计师
├── 汽车与智能交通
│   ├── 汽车工程师 → 整车架构师 → 技术总监 → CTO
│   ├── 自动驾驶工程师 → 系统架构师 → 自动驾驶总监
│   └── 新能源汽车: 电池/电机/电控工程师 → 三电总工
├── 交通规划
│   ├── 交通规划师 → 高级规划师 → 交通总工 → 城市交通局长
│   └── 轨道交通工程师 → 系统专家 → 工程总监
└── 物流运营
    ├── 物流专员 → 物流经理 → 物流总监 → COO
    └── 供应链分析师 → 供应链经理 → CSCO
```

## 🌱 第1级 · 入门

### 航空工程入门

**基础学科**
- [ ] 空气动力学：升力/阻力/伯努利原理
- [ ] 结构力学：应力应变、疲劳分析
- [ ] 推进系统：涡扇/涡桨/火箭发动机原理
- [ ] 飞行力学：稳定性和控制基础
- [ ] 适航标准：CCAR/FAR适航规章基础

**工具技能**
- [ ] CAD/CAM：CATIA / SolidWorks
- [ ] CFD仿真：ANSYS Fluent基础
- [ ] 结构分析：NASTRAN / Abaqus入门
- [ ] 复合材料基础：碳纤维/蜂窝结构

### 飞行员入门路径

```
路径一（航空公司）：
  PPL（私照）→ ATPL理论课程 → CPL（商照）→ IR（仪表）→ ME（多发）→ 航司招募（MPL/CADET）

路径二（军转民）：
  军队飞行员 → 退役转商业航空 → 民航改装训练

关键证照（中国民航）：
  PPL → CPL → ATPL（航线运输驾驶员执照）
  仪表等级（IR）+ 多发等级（ME）必须
```

### 自动驾驶工程师入门
- [ ] ROS（机器人操作系统）基础
- [ ] 传感器融合基础：相机/LiDAR/Radar
- [ ] 路径规划算法：A*、Dijkstra、RRT
- [ ] Python/C++编程（后者更重要）
- [ ] 机器学习在感知中的应用

### 关键里程碑
- ✅ 完成航空/汽车公司实习，接触真实工程项目
- ✅ 飞行员：取得PPL（私人飞行员执照）
- ✅ 自动驾驶：能运行一个基础的自动驾驶仿真Demo

## 🌿 第2级 · 初级 (2-5年)

### 航空结构工程师
- [ ] 复合材料详细设计：铺层顺序、工艺选择
- [ ] 疲劳/损伤容限（F&DT）分析
- [ ] 制造工艺：RTM/热压罐/自动铺带（ATL/AFP）
- [ ] 试验验证：地面强度试验、静强度试验
- [ ] DO-160 / CS-25 适航符合性验证

### 飞行员进阶
- [ ] 取得CPL + IR + ME（多发仪表商照）
- [ ] 航空公司MPL培训或外聘招募
- [ ] 副机长资格：机型改装→路线检查→放飞
- [ ] CRM（机组资源管理）：团队协作和决策
- [ ] UPRT（非正常姿态恢复）训练

### 自动驾驶进阶

**感知方向**
- [ ] 目标检测：YOLO、Transformer-based检测
- [ ] 3D点云处理：PointNet、PointPillars
- [ ] 多传感器融合：卡尔曼滤波、深度融合

**规划与控制方向**
- [ ] 行为预测：轨迹预测模型
- [ ] 运动规划：Frenet坐标系规划
- [ ] 控制算法：MPC（模型预测控制）
- [ ] 高精地图：HD Map的生产和消费

**仿真验证**
- [ ] CARLA / SUMO仿真平台
- [ ] 场景库构建：corner case挖掘
- [ ] 安全性验证方法：形式化验证、统计测试

### 物流/供应链初级
- [ ] 仓储管理：WMS系统（库位规划、收发货流程）
- [ ] 运输管理：TMS系统，承运商管理
- [ ] 最后一公里：城市配送、路径优化算法
- [ ] 3PL/4PL管理：外包物流管理

### 关键里程碑
- ✅ 飞行员：取得航线运输机型签注，独立执飞航线
- ✅ 自动驾驶：主导一个模块（感知/规划/控制）的开发
- ✅ 物流：独立管理一个仓库或运输网络节点

## 🌳 第3级 · 中级 (5-10年)

### 航空系统工程师

**系统集成**
- [ ] ARP 4754A：民机系统开发标准
- [ ] DO-178C / DO-254：机载软件/硬件适航
- [ ] 系统安全分析：FHA/FMEA/FTA
- [ ] 电传操控（FBW）系统理解
- [ ] 航电系统集成：IMA（综合模块化航电）

**项目管理**
- [ ] EVMS（挣值管理）：进度/成本管控
- [ ] 适航取证项目：与CAAC沟通和TC取证
- [ ] 供应商管理：AS9100质量体系

### 机长级飞行员

- [ ] 机长资格取得：机长检查→监控机长→正式机长
- [ ] TRI/TRE培训资质（有意向培训别人）
- [ ] 航务管理：参与航司安全委员会

### 自动驾驶总监预备

- [ ] 全栈自动驾驶系统设计：感知→预测→规划→控制→执行
- [ ] L4级功能开发主导（特定场景）
- [ ] 法规理解：GB/T 34590、ISO 26262（功能安全）
- [ ] 量产经验：从实验室到量产车的整个过程

### 关键里程碑
- ✅ 主持一个适航取证项目或子系统取证
- ✅ 自动驾驶：成功将L4功能在特定场景量产落地
- ✅ 带领10+人团队

## 🤖 AI时代的航空航天与交通规划

### AI对航空航天的颠覆性影响

```
已在落地：
  ✅ AI飞行辅助：自动驾驶仪AI增强（异常态检测/恢复建议）
  ✅ 预测性维护：发动机/机体的AI健康监测（美国航空已大规模使用）
  ✅ 航线优化：AI实时优化航路（节省燃油3-5%）
  ✅ AI适航审查：文件审查AI辅助（加速取证周期）
  ✅ 卫星图像AI分析：军事侦察/气象/灾害监测

颠覆性应用（未来5年）：
  🔥 自动驾驶航空器（eVTOL）：城市空中交通（UAM）
  🔥 无人运货机：货运无人机开始商业化（UPS/Amazon）
  🔥 AI设计工具：生成式设计颠覆传统航空研发流程
  🔥 太空商业化：SpaceX模式+AI生产提速
```

### 各角色AI融合建议

**航空工程师 — AI是设计加速器**
```
AI辅助设计（立即可用）：
  - 生成式设计：Autodesk Fusion + AI生成拓扑优化结构
  - AI仿真：Physics-informed Neural Networks加速CFD计算
  - AI辅助适航文件：Claude帮你梳理适航要求、检查文件完整性
  - Python数据分析：飞行测试数据的AI辅助分析

新技能要求：
  → 机器学习基础（理解预测性维护模型的原理）
  → 数字孪生概念（如何构建飞机的数字镜像）
  → AI适航验证：如何证明AI系统满足DO-178C安全要求
```

**自动驾驶工程师 — 站在AI浪潮最前沿**
```
2026年自动驾驶格局：
  - L4 Robotaxi：Waymo / 百度萝卜快跑已商业运营
  - L2++：特斯拉FSD / 华为ADS 2.0 大规模量产
  - L4货运：干线自动驾驶卡车正在商业落地

技术热点（必须追踪）：
  - 端到端自动驾驶：用一个大模型替代感知+规划+控制
  - 世界模型：类GPT的自动驾驶基础模型
  - 数据飞轮：真实数据+仿真数据的混合训练
  - VLA（视觉语言行动模型）：自然语言理解+驾驶决策

职业建议：
  ✅ 专注端到端/世界模型方向（高价值、高门槛）
  ✅ 建立仿真测试专长（数据驱动验证是关键瓶颈）
  ⚠️ 传统规则based规划正在被神经网络取代，尽快转型
```

**飞行员 — AI时代飞行员的角色重定位**
```
现实评估：
  - 商业航空的自动化程度已经很高（90%+飞行时间自动驾驶）
  - AI飞行员技术在快速发展，货运无人机已商业化
  - 短期内（10年）：商业客运仍需认证飞行员
  - 中期（10-20年）：单飞行员驾驶舱可能成为现实

飞行员的转型建议：
  ✅ 理解和监督AI系统：熟悉自动驾驶仪、ATMS的AI模块
  ✅ 异常情况处理专家：AI不能处理的极端情况，需要你
  ✅ 训练方向转型：教飞行员如何与AI协作的教员
  ✅ 无人机领域：考取BVLOS（超视距）无人机飞行资质
  ✅ 运营管理：转向航务/运控/飞行安全管理岗位
```

**物流/供应链 — AI驱动最后一公里革命**
```
AI物流应用（爆发中）：
  - AI路径优化：实时交通+订单量预测的动态路径规划
  - 自动分拣：机器人+AI视觉的自动分拣（菜鸟/京东仓库）
  - 需求预测：AI预测各区域订单量，提前调配库存
  - 无人配送：无人车/无人机最后一公里（已试点）

物流人员的AI转型：
  → 学习TMS/WMS系统的AI功能模块
  → 掌握基础数据分析（物流KPI的AI辅助分析）
  → 理解物流自动化设备（AGV/分拣机器人）的运营逻辑
  → 向"人机协作调度"岗位转型（管理AI和机器人的人）
```
